`asr-data` 用于构建和管理 ASR 数据集，同时提供 Rust 和 Python API。它以
[`Audio`](/asr-data/api#audio) 为中心，统一组织音频来源、按声道时间轴、参
考标注与模型预测，并通过 SQLite 完成持久化和查询。

源码：[di-osc/asr-data](https://github.com/di-osc/asr-data)

## 安装 {id="installation"}

Python 版本要求为 3.10 或更高：

```bash
pip install asr-data
```

Rust 项目可以直接添加 crate：

```bash
cargo add asr-data
```

## 核心概念 {id="concepts"}

下图展示了从音频来源、运行时波形到时间轴标注和数据库持久化的完整关系：

<Image
  src="/asr-data/architecture.svg"
  alt="asr-data 从 AudioSource 到 Audio、Waveform、AudioChunk、Timeline 与 TimeSpan 的数据模型关系图"
  width="1200"
/>

- **音频来源**：[`AudioSource`](/asr-data/api#audio-source) 统一表达
  Path、URL、Bytes、Base64 和 PCM。调用 `probe()` / `aprobe()` 得到不含样本的
  [`AudioInfo`](/asr-data/api#audio-info)。`open()` 创建延迟解码的
  [`Audio`](/asr-data/api#audio)，`load()` 则创建已加载完整波形的 `Audio`。
- **中心音频对象**：[`Audio`](/asr-data/api#audio) 同时拥有来源、`info`、稳定
  ID、业务元数据和自动创建的各声道 [`Timeline`](/asr-data/api#timeline)。
  `as_waveform()` 在需要时返回完整 [`Waveform`](/asr-data/api#waveform)；
  `stream()` / `astream()` 按顺序产生 [`AudioChunk`](/asr-data/api#audio-chunk)。
- **波形视图**：[`Waveform`](/asr-data/api#waveform) 是可重采样、切片和声道转换
  的浮点样本；`Timeline.as_waveform()` 与 `TimeSpan.as_waveform()` 会直接返回对应
  声道或时间范围的波形，不需要模型额外接收 `AudioSource`。
- **Timeline 与 TimeSpan**：每个 [`Timeline`](/asr-data/api#timeline) 具有
  channel 和 duration，并将标注分为不带来源的 `Reference` 与必须提供
  `source` 的 `Prediction`。
- **TimeSpan 与 Annotation**：[`TimeSpan`](/asr-data/api#time-span) 保存
  `start_ms`、`end_ms`、`confidence` 和 `source`，其 `annotation` 可以是
  [`AudioActivity`](/asr-data/api#audio-activity)、[`Speaker`](/asr-data/api#speaker) 或
  [`Transcription`](/asr-data/api#transcription)。[`Speaker`](/asr-data/api#speaker)
  使用 `name` 标识说话人，并可包含
  [`Transcription`](/asr-data/api#transcription)；[`Transcription`](/asr-data/api#transcription)
  保存 text、language、confidence 以及细粒度 [`Token`](/asr-data/api#token)。
- **数据库**：[`AudioDB`](/asr-data/api#audio-db) 持久化 `Audio` 的来源、信息、
  metadata、timeline 和标注；运行时波形不写入数据库，读取后仍可按需加载。

## 快速开始 {id="quickstart"}

下面的示例创建一条音频记录，加入参考说话人标注和模型预测，然后写入数据库：

```python
from asr_data import AudioDB, AudioSource
from asr_data.annotation import AudioActivity, Speaker, Transcription

# 创建来源
URL = "https://deepasset.oss-cn-beijing.aliyuncs.com/example.wav"
audio = AudioSource.from_url(URL).open(id="example-001")
audio.metadata["split"] = "train"

# 获取自动创建的时间线
timeline = audio.timeline("mono")
end_ms = timeline.duration_ms

# 添加声音活动标注；event 为空时表示类别未知
timeline.reference.annotate_span(
    0,
    end_ms,
    AudioActivity(event="speech"),
)

# 添加说话人标注
timeline.reference.annotate_span(
    0,
    end_ms,
    Speaker(
        "speaker-1",
        transcription=Transcription("甚至出现交易几乎停滞的情况", language="zh"),
    ),
)

# 添加模型预测
timeline.prediction.annotate_span(
    0,
    end_ms,
    Transcription("甚至出现交易几乎停滞的情况", language="zh", confidence=0.96),
    source="qwen-asr",
)

# 数据库
db = AudioDB.create("dataset.db")
if audio.id in db:
    db.delete(audio.id)
db.insert(audio)
loaded = db[audio.id]
loaded_timeline = loaded.timeline("mono")
print(loaded_timeline.reference.transcript().text)
print(loaded_timeline.prediction.transcript("qwen-asr").text)
```

<Infobox title="时间单位">

所有 annotation 范围和 timeline 时长均使用毫秒。添加 annotation 时必须满足
`0 <= start_ms <= end_ms <= timeline.duration_ms`。

</Infobox>
