`asr-data` 用于构建和管理 ASR 数据集，同时提供 Rust 和 Python API。它以
[`AudioDoc`](/asr-data/api#audio-doc) 为中心，统一组织音频来源、按声道时间轴、参
考标注与模型预测，并通过 SQLite 完成持久化和查询。

源码：[di-osc/asr-data](https://github.com/di-osc/asr-data)

## 安装 {id="installation"}

Python 版本要求为 3.10 或更高：

```bash
pip install asr-data
```

Rust 项目可以直接添加 crate：

```bash
cargo add asr-data
```

## 核心概念 {id="concepts"}

下图展示了从音频来源、解码结果到文档标注和数据库持久化的完整关系：

<Image
  src="/asr-data/architecture-hd.png"
  alt="asr-data 从 AudioSource 到 Audio、AudioChunk、AudioDoc、Timeline 与 Annotation 的数据模型关系图"
  width="1000"
/>

- **音频来源**：[`AudioSource`](/asr-data/api#audio-source) 统一表达
  Path、URL、Bytes、Base64 和 PCM。调用 `load()` / `aload()` 得到完整的
  [`Audio`](/asr-data/api#audio)，调用 `stream()` / `astream()` 则持续产生
  [`AudioChunk`](/asr-data/api#audio-chunk)。
- **解码后的音频**：[`Audio`](/asr-data/api#audio) 保存浮点 `Samples` 和
  `AudioFormat`；[`AudioChunk`](/asr-data/api#audio-chunk) 额外记录 `offset_ms`
  与 `is_final`。两者都支持声道提取、重采样和切片。
- **可持久化文档**：[`AudioDoc`](/asr-data/api#audio-doc) 引用原始
  [`AudioSource`](/asr-data/api#audio-source)，并保存稳定 ID、业务元数据和各声道
  的 [`Timeline`](/asr-data/api#timeline)。解码后的
  [`Audio`](/asr-data/api#audio) 与 [`AudioChunk`](/asr-data/api#audio-chunk) 不
  会直接存入 [`AudioDoc`](/asr-data/api#audio-doc)。
- **Timeline 与 Annotation**：每个 [`Timeline`](/asr-data/api#timeline) 具有
  channel 和 duration，并将 annotation 分为不带来源的 `Reference` 与必须提供
  `source` 的 `Prediction`。
- **Annotation payload**：[`Annotation`](/asr-data/api#annotation) 保存
  range 和 confidence，其 payload 可以是
  `Speech`、[`Speaker`](/asr-data/api#speaker) 或
  [`Transcription`](/asr-data/api#transcription)。[`Speaker`](/asr-data/api#speaker)
  使用 `name` 标识说话人，并可包含
  [`Transcription`](/asr-data/api#transcription)；[`Transcription`](/asr-data/api#transcription)
  保存 text、language、confidence 以及细粒度 [`Token`](/asr-data/api#token)。
- **数据库**：[`AudioDB`](/asr-data/api#audio-db) 将完整的
  [`AudioDoc`](/asr-data/api#audio-doc) 持久化到可查询的 SQLite 数据库。

## 快速开始 {id="quickstart"}

下面的示例创建一条音频记录，加入参考说话人标注和模型预测，然后写入数据库：

```python
from asr_data import AudioDB, AudioDoc, AudioSource
from asr_data.annotation import Speaker, Transcription

# 加载音频
URL = "https://deepasset.oss-cn-beijing.aliyuncs.com/example.wav"
source = AudioSource.from_url(URL)
audio = source.load()

# 创建音频文档
doc = AudioDoc(source, id="example-001")
doc.metadata["split"] = "train"

# 创建时间线用于标注
timeline = doc.ensure_timeline("mono", duration_ms=audio.duration_ms)
end_ms = timeline.duration_ms

# 添加说话人标注
timeline.reference.add_speaker(
    0,
    end_ms,
    Speaker(
        "speaker-1",
        transcription=Transcription("甚至出现交易几乎停滞的情况", language="zh"),
    ),
)

# 添加模型预测
timeline.prediction.add_transcription(
    0,
    end_ms,
    Transcription("甚至出现交易几乎停滞的情况", language="zh", confidence=0.96),
    source="qwen-asr",
)

# 数据库
db = AudioDB("dataset.db")
if doc.id in db:
    db.delete(doc.id)
db.insert(doc)
loaded = db[doc.id]
loaded_timeline = loaded.timeline("mono")
print(loaded_timeline.reference.transcript().text)
print(loaded_timeline.prediction.transcript("qwen-asr").text)
```

<Infobox title="时间单位">

所有 annotation 范围和 timeline 时长均使用毫秒。添加 annotation 时必须满足
`0 <= start_ms <= end_ms <= timeline.duration_ms`。

</Infobox>
