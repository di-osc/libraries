## AudioSource {id="audio-source",tag="class"}

描述尚未解码的音频来源。来源可以是文件路径、URL、编码字节、base64 字符串或 PCM
S16LE 字节。`load()` 解码为完整 [`Audio`](#audio)，`stream()` 创建逐块解码的
[`AudioStream`](#audio-stream)。

## AudioSource.from\_\* {id="audio-source-from",tag="class methods"}

从路径、URL、编码字节、base64 或 PCM 创建尚未解码的音频来源。构造时通常不会读取
文件、发起网络请求或解码样本；真正的 I/O 发生在 `probe()`、`load()` 或
`stream()` 时。`from_pcm()` 没有文件头，因此必须显式提供 `sample_rate` 和
`channels`。

> #### Example
>
> ```python
> import base64
> from pathlib import Path
> from tempfile import TemporaryDirectory
> from urllib.request import urlopen, urlretrieve
>
> import numpy as np
>
> from asr_data import AudioSource, Waveform
>
> URL = "https://deepasset.oss-cn-beijing.aliyuncs.com/example.wav"
>
> with TemporaryDirectory() as directory:
>     path = Path(directory) / "example.wav"
>     urlretrieve(URL, path)
>     from_path = AudioSource.from_path(str(path))
>
> from_url = AudioSource.from_url(URL)
> with urlopen(URL) as response:
>     audio_bytes = response.read()
> from_bytes = AudioSource.from_bytes(audio_bytes)
> from_base64 = AudioSource.from_base64(base64.b64encode(audio_bytes).decode("ascii"))
>
> waveform = Waveform.from_url(URL).to_mono().resample(16_000)
> pcm_bytes = (np.clip(waveform.samples, -1.0, 1.0) * 32_767).astype("<i2").tobytes()
> from_pcm = AudioSource.from_pcm(pcm_bytes, sample_rate=16_000, channels=1)
>
> for source in (from_path, from_url, from_bytes, from_base64, from_pcm):
>     audio = source.load()
>     print(source.kind, audio.info.sample_rate, audio.info.channels)
> ```

| Method | Input |
| --- | --- |
| `from_path(path)` | 本地音频文件路径。~~str~~ |
| `from_url(url)` | HTTP 或 HTTPS 音频地址。~~str~~ |
| `from_bytes(data)` | WAV、MP3 等编码音频字节。~~bytes~~ |
| `from_base64(data)` | base64 编码的音频内容。~~str~~ |
| `from_pcm(data, sample_rate, channels=1)` | PCM S16LE 字节。~~bytes, int, int~~ |

所有方法均返回新的 [`AudioSource`](#audio-source)。

## AudioSource.load {id="audio-source-load",tag="method"}

完整读取并解码来源，创建带波形、音频信息和各声道时间轴的 [`Audio`](#audio)。
接受可选的关键字参数 `id`。

> #### Example
>
> ```python
> from asr_data import AudioSource
>
> URL = "https://deepasset.oss-cn-beijing.aliyuncs.com/example.wav"
> source = AudioSource.from_url(URL)
> audio = source.load(id="example-001")
> print(audio.id, audio.info.duration_ms)
> waveform = audio.as_waveform().to_mono().resample(16_000)
> print(waveform.sample_rate, waveform.channels)
> ```

| Name           | Description                    |
| -------------- | ------------------------------ |
| _keyword-only_ |                                |
| `id`           | 可选稳定 ID。~~Optional[str]~~ |
| **RETURNS**    | 中心音频对象。~~Audio~~        |

异步程序使用 `await source.aload(...)`。

## AudioSource.stream {id="audio-source-stream",tag="method"}

创建与 [`Audio`](#audio) 同级的 [`AudioStream`](#audio-stream)，按固定目标时长
逐块解码。创建流时只探测格式，不读取完整波形。

> #### Example
>
> ```python
> from asr_data import AudioSource
>
> URL = "https://deepasset.oss-cn-beijing.aliyuncs.com/example.wav"
> stream = AudioSource.from_url(URL).stream(chunk_size_ms=100, id="call-001")
>
> for chunk in stream:
>     print(chunk.offset_ms, stream.position_ms, stream.timeline("mono").duration_ms)
>
> audio = stream.to_audio()
> ```

| Name            | Description                                        |
| --------------- | -------------------------------------------------- |
| `chunk_size_ms` | 每块的目标时长，默认为 `100`。~~int~~              |
| _keyword-only_  |                                                    |
| `id`            | 可选稳定 ID。~~Optional[str]~~                     |
| **RETURNS**     | 可同步或异步消费的音频流。~~AudioStream~~          |

同一个 `AudioStream` 可以使用普通 `for` 或 `async for`，但一次流实例不能混合同步与
异步消费。

## AudioStream {id="audio-stream",tag="class"}

逐步增长的流式音频文档，由 [`AudioSource.stream()`](#audio-source-stream) 创建。
它与 [`Audio`](#audio) 处于同一层级，提供相同的 `id`、`source`、`info`、
`metadata`、`timelines`、`timeline()` 和 `as_waveform()` 访问方式；区别是波形和
时间轴只覆盖已经迭代到的音频。

每次取得 [`AudioChunk`](#audio-chunk) 前，该块会先追加到流中。因此模型处理当前
chunk 时，可以读取包括当前块在内的累计波形，并给任意已接收历史范围添加标注；
超过 `position_ms` 的未来范围会被拒绝。

`AudioStream` 提供与 [`AudioSource`](#audio-source) 对称的便捷构造器，默认
`chunk_size_ms=100`，都接受 keyword-only 的可选 `id`：

| Method | Input |
| --- | --- |
| `from_path(path, ...)` | 本地文件路径 |
| `from_url(url, ...)` | HTTP、HTTPS 或 file URL |
| `from_bytes(data, ...)` | WAV、MP3 等编码音频字节 |
| `from_base64(data, ...)` | 编码音频的 base64 字符串 |
| `from_pcm(data, sample_rate, channels=1, ...)` | PCM S16LE 原始字节 |

> #### Example
>
> ```python
> from asr_data import AudioStream
> from asr_data.annotation import Transcription
>
> URL = "https://deepasset.oss-cn-beijing.aliyuncs.com/example.wav"
> stream = AudioStream.from_url(URL, chunk_size_ms=1_000, id="call-001")
>
> for chunk in stream:
>     timeline = chunk.timeline("mono")
>     print(stream.position_ms, timeline.duration_ms)
>
> timeline.annotate_span(
>     0,
>     timeline.duration_ms,
>     Transcription("甚至出现交易几乎停滞的情况", language="zh"),
>     is_reference=True,
> )
> audio = stream.to_audio()
> ```

| Name            | Description                                                   |
| --------------- | ------------------------------------------------------------- |
| `position_ms`   | 当前已接收音频的结束位置。~~int~~                             |
| `is_complete`   | 是否已消费到来源结尾。~~bool~~                                |
| `is_closed`     | 是否已经关闭。~~bool~~                                        |
| `as_waveform()` | 返回当前累计波形。~~Waveform~~                                |
| `display(...)`  | 在 Jupyter 中播放当前累计波形。~~None~~                       |
| `timeline()`    | 返回指定声道的增长中时间轴。~~Optional[Timeline]~~            |
| `to_audio()`    | 完整消费后零重复解码地转换为 `Audio`。~~Audio~~               |
| `close()`       | 关闭流；关闭后不能继续迭代或转换。~~None~~                    |

`async for chunk in stream` 使用同一组属性和方法。一个实例一旦开始同步或异步消费，
后续就必须保持该模式。`to_audio()` 只接受完整流，并保留累计波形、时间轴、标注和
metadata。

## AudioChunk {id="audio-chunk",tag="class"}

流式解码产生的音频块。它提供与 [`Waveform`](#waveform) 相同的样本、格式以及
`to_mono()`、 `channel()`、`resample()`、`slice_ms()` 变换，并额外提供以下属性。

| Name        | Description                         |
| ----------- | ----------------------------------- |
| `offset_ms` | 当前块相对原音频的起始偏移。~~int~~ |
| `end_ms`    | 当前块在完整音频中的结束位置。~~int~~ |
| `is_final`  | 是否为流中的最后一块。~~bool~~      |

`as_waveform(channel=None)` 返回当前块的波形；`timeline(channel)` 返回父
[`AudioStream`](#audio-stream) 当前增长到的全局时间轴；
`to_global_span(start_ms, end_ms)` 把块内相对范围转换为流中的全局范围。
`display(start_ms=None, end_ms=None, autoplay=False)` 直接播放当前 chunk，时间参数
使用 chunk 内的局部坐标。

## Audio {id="audio",tag="class"}

完整音频、时间轴与标注的中心对象。它包含稳定 ID、[`AudioSource`](#audio-source)、
[`AudioInfo`](#audio-info)、业务元数据、各声道 [`Timeline`](#timeline) 和完整
运行时 [`Waveform`](#waveform)。数据库不会持久化浮点样本。

## Audio.\_\_init\_\_ {id="audio-init",tag="method"}

从来源创建完整音频记录。构造时加载波形、保存 `info`，并自动创建全部原始声道的
timeline。通常优先使用 [`AudioSource.load()`](#audio-source-load)。

> #### Example
>
> ```python
> from asr_data import AudioSource
>
> URL = "https://deepasset.oss-cn-beijing.aliyuncs.com/example.wav"
> audio = AudioSource.from_url(URL).load(id="call-001")
> print(audio.id, audio.source.url)
> print(audio.info.duration_ms, audio.timelines.keys())
> ```

| Name     | Description                                    |
| -------- | ---------------------------------------------- |
| `source` | 音频来源。~~AudioSource~~                      |
| `id`     | 稳定业务 ID；省略时自动生成。~~Optional[str]~~ |

单声道创建 `mono`；双声道创建 `left` 和 `right`；更多声道继续以 `2`、`3` 等
编号。每条 timeline 的整数毫秒时长由 `info` 向上取整得到。异步程序使用
`await source.aload(id=...)`。

| Name          | Description                                         |
| ------------- | --------------------------------------------------- |

| `duration_ms` | Timeline 总时长。~~int~~                            |
| `reference`   | 不带 source 的参考标注。~~ReferenceSpans~~         |
| `prediction`  | 必须带 source 的预测标注。~~PredictionSpans~~      |

`as_waveform()` 返回该声道的完整波形；`display()` 可直接播放该声道的全部或部分
音频。`annotate_span()` 会立即校验
`0 <= start_ms <= end_ms <= timeline.duration_ms`；内容完全相同的重复添加会返回
已有 `TimeSpan`，不会插入第二条记录。

## Timeline.eval {id="timeline-eval",tag="method"}

使用 reference 评估 prediction。不传 source 时自动评测全部可用来源；也可以为转写
和 Activity 分别指定一个或多个来源。转写默认先执行内嵌中文 TN，再计算 CER。

> #### Example
>
> ```python
> from asr_data import AudioSource
> from asr_data.annotation import AudioActivity, Transcription
>
> URL = "https://deepasset.oss-cn-beijing.aliyuncs.com/example.wav"
> doc = AudioSource.from_url(URL).load()
> timeline = doc.timeline("mono")
> text = "甚至出现交易几乎停滞的情况"
>
> timeline.annotate_span(
>     0, timeline.duration_ms, AudioActivity(event="speech"), is_reference=True
> )
> timeline.annotate_span(
>     0, timeline.duration_ms, Transcription(text, language="zh"), is_reference=True
> )
> timeline.annotate_span(
>     0,
>     timeline.duration_ms,
>     AudioActivity(event="speech"),
>     is_reference=False,
>     source="silero-vad",
> )
> timeline.annotate_span(
>     0,
>     timeline.duration_ms,
>     Transcription(text, language="zh"),
>     is_reference=False,
>     source="qwen-asr",
> )
>
> result = timeline.eval()
> print(
>     result.transcription["qwen-asr"].cer,
>     result.activity["silero-vad"].f1,
> )
> ```

| Name            | Description                                                   |
| --------------- | ------------------------------------------------------------- |
| _keyword-only_  |                                                               |
| `transcription` | 转写来源或来源列表；省略时自动发现。~~Optional[str \| List[str]]~~ |
| `activity`      | Activity 来源或来源列表；省略时自动发现。~~Optional[str \| List[str]]~~ |
| `normalize`     | 是否执行内嵌中文 TN，默认为 `True`。~~bool~~                  |
| **RETURNS**     | 按任务和 source 分组的结果。~~TimelineEvaluation~~            |

不传两个 source 参数时自动评测全部具有 reference 的任务。只传一个参数时只评测该
任务。显式指定的 reference 或 prediction 不存在，以及完全没有可评测内容时，抛出
[`AsrDataError`](#asr-data-error)。

## ReferenceSpans {id="reference-spans",tag="class"}

参考真值只读集合；新标注统一通过 [`Timeline.annotate_span()`](#timeline-annotate-span)
写入。Reference annotation 的 `source` 始终为 `None`。相同 event 的
Activity 之间不得重叠；不同 event 可以重叠。同名 Speaker 之间不得重叠；顶层
Transcription 之间不得重叠，且不得与携带 transcription 的 Speaker 重叠。时间范围
采用 `[start_ms, end_ms)`，首尾相接不算重叠。完整规则见
[标注校验规范](/asr-data/annotations#validation)。

## Timeline.annotate_span {id="timeline-annotate-span",tag="method"}

Timeline 唯一的标注写入口。`is_reference=True` 写入参考答案，此时必须省略
`source`；`is_reference=False` 写入模型预测，此时必须提供非空 `source`。
第三个参数接受 [`AudioActivity`](#audio-activity)、[`Speaker`](#speaker)、
[`Transcription`](#transcription) 或 [`Token`](#token)，不接受纯字符串。

> #### Example
>
> ```python
> from asr_data import Audio
> from asr_data.annotation import AudioActivity, Speaker, Transcription
>
> URL = "https://deepasset.oss-cn-beijing.aliyuncs.com/example.wav"
> timeline = Audio.from_url(URL).timeline("mono")
>
> activity = timeline.annotate_span(
>     0,
>     timeline.duration_ms,
>     AudioActivity(event="speech", confidence=0.98),
>     is_reference=True,
> )
> prediction = timeline.annotate_span(
>     0,
>     timeline.duration_ms,
>     Speaker(
>         "agent",
>         transcription=Transcription("甚至出现交易几乎停滞的情况"),
>     ),
>     is_reference=False,
>     source="diarization-v2",
> )
> print(activity.annotation.event, prediction.source)
> ```

| Name             | Description                                                     |
| ---------------- | --------------------------------------------------------------- |
| `start_ms`       | 起始时间。~~int~~                                               |
| `end_ms`         | 结束时间，不得超过 timeline duration。~~int~~                   |
| `annotation`     | 具体标注 payload。~~Annotation~~                                |
| _keyword-only_   |                                                                 |
| `is_reference`   | 是否写入参考答案。~~bool~~                                      |
| `source`         | Prediction 的模型或流程；reference 必须省略。~~Optional[str]~~  |
| **RETURNS**      | 新建或去重后已有的标注句柄。~~TimeSpan~~                        |

`timeline.reference` 和 `timeline.prediction` 只负责读取、筛选、组合 transcript
以及来源管理，不提供第二套标注写方法。带时间范围的 token 必须完全位于所属
Transcription 或 Speaker annotation 的范围内。

## TimeSpan.annotation / as_waveform {id="time-span-annotation",tag="property"}

获取或整体替换标注内容。新 annotation 必须与当前内容类型一致。替换
AudioActivity、Transcription 或 Speaker 时会重新校验 event、token 时间范围和重叠
规则；校验失败时原内容保持不变。`as_waveform()` 返回当前 `[start_ms, end_ms)`
对应的波形，`display()` 直接播放该范围，传入的起止时间使用 TimeSpan 内的局部
坐标。完整规则见
[标注校验规范](/asr-data/annotations#validation)。

> #### Example
>
> ```python
> from asr_data import AudioSource
> from asr_data.annotation import Speaker

> URL = "https://deepasset.oss-cn-beijing.aliyuncs.com/example.wav"
> audio = AudioSource.from_url(URL).load()
> timeline = audio.timeline("mono")
> span = timeline.annotate_span(
>     0, timeline.duration_ms, Speaker("agent"), is_reference=True
> )
>

> ```

| **RETURNS** | 当前标注内容。~~Annotation~~ |

## AudioDB.query {id="audio-db-query",tag="method"}

按 ID 游标、时长、数据库自动维护的创建/修改时间和元数据筛选记录。

> #### Example
>
> ```python
> from datetime import datetime, timedelta, timezone
> from pathlib import Path
> from tempfile import TemporaryDirectory
>
> from asr_data import AudioDB, Audio, AudioSource
>
> URL = "https://deepasset.oss-cn-beijing.aliyuncs.com/example.wav"
>
> with TemporaryDirectory() as directory:
>     path = Path(directory) / "dataset.db"
>     db = AudioDB.create(str(path))
>     doc = AudioSource.from_url(URL).load(id="call-001")
>     doc.metadata["split"] = "train"
>     db.insert(doc)
>
>     now = datetime.now(timezone.utc)
>     page = db.query(
>         limit=100,
>         created_from=now - timedelta(days=7),
>         updated_from=now - timedelta(hours=24),
>         metadata={"split": "train"},
>     )
>     print([item.id for item in page])
> ```

| Name              | Description                                          |
| ----------------- | ---------------------------------------------------- |
| `limit`           | 最大返回数，默认为 `100`，上限 `10_000`。~~int~~     |
| _keyword-only_    |                                                      |
| `after`           | 上一页最后一条 audio ID。~~Optional[str]~~           |
| `min_duration_ms` | 可选最短时长。~~Optional[int]~~                      |
| `max_duration_ms` | 可选最长时长。~~Optional[int]~~                      |
| `created_from`    | 创建时间下界，包含。~~Optional[datetime]~~           |
| `created_until`   | 创建时间上界，不包含。~~Optional[datetime]~~         |
| `updated_from`    | 修改时间下界，包含。~~Optional[datetime]~~           |
| `updated_until`   | 修改时间上界，不包含。~~Optional[datetime]~~         |
| `metadata`        | 精确匹配的 JSON 元数据。~~Optional[Dict[str, Any]]~~ |
| **RETURNS**       | 按 audio ID 排序的记录。~~List[Audio]~~           |

时间参数必须是带时区的 `datetime`，并使用 `from <= timestamp < until` 半开区间。
创建和修改时间由 `AudioDB` 自动维护，不会出现在返回的 `Audio` 中。

## AudioDB.eval {id="audio-db-eval",tag="method"}

评测数据库中全部匹配记录。内部通过 Audio ID 游标分页，`batch_size` 只影响每批
读取数量，不限制最终数据集大小。

> #### Example
>
> ```python
> from tempfile import TemporaryDirectory
>
> from asr_data import AudioDB, Audio, AudioSource
> from asr_data.annotation import Transcription
>
> URL = "https://deepasset.oss-cn-beijing.aliyuncs.com/example.wav"
> text = "甚至出现交易几乎停滞的情况"
>
> with TemporaryDirectory() as directory:
>     db = AudioDB.create(f"{directory}/evaluation.db")
>     doc = AudioSource.from_url(URL).load(id="sample-1")
>     timeline = doc.timeline("mono")
>     timeline.annotate_span(
>         0, timeline.duration_ms, Transcription(text), is_reference=True
>     )
>     timeline.annotate_span(
>         0, timeline.duration_ms, Transcription(text),
>         is_reference=False, source="qwen-asr"
>     )
>     doc.metadata["split"] = "test"
>     db.insert(doc)
>
>     result = db.eval(
>         transcription="qwen-asr",
>         metadata={"split": "test"},
>         batch_size=100,
>     )
>     print(result.transcription["qwen-asr"].cer)
>     print(result.transcription["qwen-asr"].coverage)
> ```

| Name              | Description                                                   |
| ----------------- | ------------------------------------------------------------- |
| _keyword-only_    |                                                               |
| `transcription`   | 转写来源或来源列表；省略时自动发现。~~Optional[str \| List[str]]~~ |
| `activity`        | Activity 来源或来源列表；省略时自动发现。~~Optional[str \| List[str]]~~ |
| `normalize`       | 是否执行中文 TN，默认为 `True`。~~bool~~                      |
| `batch_size`      | 每次读取的文档数，默认为 `100`。~~int~~                       |
| `after`           | 可选起始游标。~~Optional[str]~~                               |
| `min_duration_ms` | 可选最短时长。~~Optional[int]~~                               |
| `max_duration_ms` | 可选最长时长。~~Optional[int]~~                               |
| `created_from`    | 创建时间下界。~~Optional[datetime]~~                          |
| `created_until`   | 创建时间上界。~~Optional[datetime]~~                          |
| `updated_from`    | 修改时间下界。~~Optional[datetime]~~                          |
| `updated_until`   | 修改时间上界。~~Optional[datetime]~~                          |
| `metadata`        | 精确匹配的 JSON 元数据。~~Optional[Dict[str, Any]]~~          |
| **RETURNS**       | 数据集级聚合结果。~~DatasetEvaluation~~                       |
