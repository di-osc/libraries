音频处理从 [`AudioSource`](/asr-data/api#audio-source) 开始。它只描述数据来自哪
里；`load()` 产生完整 [`Audio`](/asr-data/api#audio)，`stream()` 产生与其平级的
[`AudioStream`](/asr-data/api#audio-stream)。模型通过 Audio、AudioStream、
Timeline、TimeSpan 或 AudioChunk 的波形视图取得所需样本。

右侧 Example 按页面顺序连续执行，后面的示例会直接使用前面创建的变量。

## 音频来源 {id="source"}

`AudioSource` 是一个尚未解码的音频来源描述，而不是音频样本本身。创建来源时通常不
读取文件、不发起网络请求，也不执行解码；真正的 I/O 发生在 `probe()`、`load()`
或 `stream()` 时。

> #### Example
>
> ```python
> from asr_data import AudioSource
>
> URL = "https://deepasset.oss-cn-beijing.aliyuncs.com/example.wav"
> source = AudioSource.from_url(URL)
>
> print(source.kind, source.url)
> ```

将“来源”和“解码结果”分开有三个目的：

- **延迟读取**：可以先组织任务，真正需要样本时再下载和解码。
- **统一入口**：文件、URL、内存字节和实时 PCM 最终都使用相同的加载与流式接口。
- **保留来源信息**：来源可以随业务数据传递，不需要长期保存体积更大的浮点样本。

支持五种来源：

| 创建方法        | 输入内容                 | 典型用途                         |
| --------------- | ------------------------ | -------------------------------- |
| `from_path()`   | 本地文件路径             | 本地数据集和缓存文件             |
| `from_url()`    | HTTP 或 HTTPS 地址       | 对象存储、数据服务和远程样本     |
| `from_bytes()`  | WAV、MP3 等编码音频字节  | 上传文件、消息体或内存缓存       |
| `from_base64()` | 编码音频的 base64 字符串 | JSON、日志或文本协议             |
| `from_pcm()`    | PCM S16LE 原始字节       | 麦克风、WebSocket 和实时音频设备 |

`from_bytes()` 中的字节仍然带有 WAV、MP3 等容器或编码信息，解码器可以从内容识别
音频格式。`from_pcm()` 没有文件头，因此必须显式提供 `sample_rate` 和
`channels`；每个样本使用有符号 16 位小端整数。

## 探测音频 {id="probe"}

`source.probe()` 读取音频容器或 PCM 参数，返回
[`AudioInfo`](/asr-data/api#audio-info)，但不会把采样解码为浮点数组。它适合在创建
标注文档前取得原始采样率、声道数、帧数和时长。

> #### Example
>
> ```python
> info = source.probe()
>
> print(info.sample_rate, info.channels)
> print(info.frame_count, info.duration_ms)
> print(info.source_format)
> ```

| 属性            | 含义                               |
| --------------- | ---------------------------------- |
| `sample_rate`   | 原始采样率                         |
| `channels`      | 原始声道数                         |
| `frame_count`   | 每个声道的有效采样帧数             |
| `duration_ms`   | 根据帧数和采样率计算的浮点毫秒时长 |
| `source_format` | 编码、采样率和声道组成的原始格式   |

异步程序使用 `await source.aprobe()`。URL 来源仍需要读取编码数据来探测容器，但
不会把整段音频转换为浮点样本。

## 加载音频 {id="load"}

`source.load()` 完整解码来源，创建具有最终时长 timeline 的 `Audio`。异步程序
使用 `await source.aload()`，参数和返回结果一致。如果不需要复用或单独保存
AudioSource，也可以直接使用 `Audio.from_path()`、`from_url()`、`from_bytes()`、
`from_base64()` 或 `from_pcm()`。

> #### Example
>
> ```python
> from asr_data import Audio
>
> audio = Audio.from_url(URL, id="example-001")
> waveform = audio.as_waveform().to_mono().resample(16_000)
>
> print(audio.info.duration_ms, audio.timeline("mono").duration_ms)
> print(waveform.sample_rate, waveform.channels)
> print(waveform.samples.dtype, waveform.source_format)
> ```

| 对象 | 保存内容 | 典型操作 |
| --- | --- | --- |
| `Audio` | 完整波形、来源、info、ID、metadata 和最终时长 timelines | 标注、全局处理、持久化 |
| `AudioStream` | 已接收波形、来源、info、metadata 和动态增长的 timelines | 边接收边推理与标注 |
| `Waveform` | `float32` 样本、采样率、声道与原始格式 | 重采样、混声道、切片、播放 |

`Waveform` 提供以下基础信息：

| 属性            | 含义                                           |
| --------------- | ---------------------------------------------- |
| `samples`       | `float32` 样本；数值范围为 `[-1.0, 1.0]`       |
| `sample_rate`   | 每秒每个声道的采样帧数                         |
| `channels`      | 声道数                                         |
| `frame_count`   | 每个声道的采样帧数                             |
| `duration_ms`   | 根据帧数计算的时长，类型为 `float`             |
| `source_format` | 解码前检测到的编码、采样率和声道等原始格式信息 |

`duration_ms` 可能包含小数毫秒，这是采样帧数除以采样率的自然结果。只有创建
timeline 或 annotation 时，才需要按相应 API 的规则转换为毫秒整数。

## 音频处理 {id="processing"}

`Waveform` 的转换方法不会原地修改已有对象，而是返回新的 `Waveform`。这种设计可以保留原
始解码结果，并安全地从同一份音频产生不同采样率、声道或时间范围的版本。

> #### Example
>
> ```python
> processed = audio.as_waveform().to_mono().resample(16_000)
> clip = processed.slice_ms(0, 2_000)
> parts = processed.split_at_low_energy(1_000)
>
> print("processed:", processed.sample_rate, processed.channels)
> print("clip:", clip.duration_ms)
> print("parts:", [part.duration_ms for part in parts])
> ```

常用操作包括：

| 方法                                   | 作用                                      |
| -------------------------------------- | ----------------------------------------- |
| `resample(sample_rate)`                | 转换到目标采样率                          |
| `to_mono()`                            | 将所有声道混合为单声道                    |
| `channel(index)`                       | 提取一个声道，索引从 `0` 开始             |
| `slice_ms(start_ms, end_ms)`           | 按毫秒截取左闭右开的时间范围              |
| `split_at_low_energy(max_duration_ms)` | 将长音频优先拆在低能量位置                |
| `display(...)`                         | 在 IPython 或 Notebook 中播放指定时间范围 |

所有能够调用 `as_waveform()` 的对象都直接提供相同签名的 `display()`：
[`Audio`](/asr-data/api#audio)、[`AudioStream`](/asr-data/api#audio-stream)、
[`AudioChunk`](/asr-data/api#audio-chunk)、[`Timeline`](/asr-data/api#timeline) 和
[`TimeSpan`](/asr-data/api#time-span)。因此在 Notebook 中不需要先手动取出
`Waveform`：

```python
audio.display(end_ms=2_000)
timeline.display(start_ms=500, end_ms=1_500)
```

多声道样本按 frame 交错保存，`frame_count` 表示每个声道的帧数，而不是 `samples`
数组的元素总数。`slice_ms()` 和低能量切分会保留声道布局；需要单声道时应显式调用
`to_mono()`。

## 流式加载 {id="streaming"}

完整加载适合需要随机切片、全局分析或反复处理的音频；流式加载适合边接收边推理、控
制单次处理粒度或尽快得到首个结果的场景。

> #### Example
>
> ```python
> from asr_data import AudioStream
>
> stream = AudioStream.from_url(
>     URL, chunk_size_ms=500, id="example-stream"
> )
>
> for chunk in stream:
>     model_input = chunk.as_waveform("mono").resample(16_000)
>     timeline = chunk.timeline("mono")
>     print(chunk.end_ms, timeline.duration_ms, model_input.frame_count)
>
> streamed_audio = stream.to_audio()
> ```

`source.stream(chunk_size_ms=...)` 与 `AudioStream.from_*()` 都返回
`AudioStream`。同一个对象既可使用普通
`for`，也可使用 `async for`，但一次流不能混用两种消费方式。`chunk_size_ms` 是
目标分块时长，最后一块可能更短。

AudioStream 创建时 timeline 时长为 `0`。每次迭代会先追加样本、把所有 timeline
延长到 `chunk.end_ms`，再返回 chunk，因此模型可以立即为当前块或任何已经到达的
历史范围添加标注。尚未接收的未来范围仍会被范围校验拒绝。

`stream.as_waveform()` 返回当前累计波形；完整消费后 `is_complete` 为 `True`，
此时 `to_audio()` 会复用已经解码的样本、metadata、timeline 和标注创建 `Audio`，
不会重新读取来源。提前 `close()` 会令 `is_closed` 为 `True`，且不能调用
`to_audio()`。

每个 `AudioChunk` 包含：

- `id`、`source`、`metadata` 和 `timelines`：与父 `AudioStream` 共享的文档上下文。
- `index`：本块在流中从 `0` 开始的顺序。
- `info`：只描述当前块的采样率、声道、帧数、时长和来源格式。
- `offset_ms`：本块相对整段音频起点的偏移。
- `end_ms`：本块在整段音频中的全局结束位置。
- `duration_ms`：本块的实际时长。
- `is_final`：是否为该来源的最后一块。

chunk 本身不重复提供 `samples`、`resample()`、`slice_ms()` 等 Waveform API。
需要当前块样本或音频变换时，先调用 `chunk.as_waveform()`。

`AudioChunk.to_timeline_range()` 可以把模型返回的 chunk 内相对范围换算为全局范围；
`chunk.timeline(channel)` 与父 AudioStream 返回同一个正在增长的 timeline，而不是
只覆盖当前 chunk 的临时时间轴。已返回 chunk 的波形和 `info` 始终固定，但它持有的
metadata 和 timeline 是共享的，因此之后的迭代与标注仍然对旧 chunk 可见。这样流式
模型可以为过去的一段时间补写标注。
