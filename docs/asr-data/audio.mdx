音频处理从 [`AudioSource`](/asr-data/api#audio-source) 开始。它只描述数据来自哪
里；`open()` / `load()` 创建中心 [`Audio`](/asr-data/api#audio)，模型再通过
`Audio.as_waveform()`、`Timeline.as_waveform()` 或 `TimeSpan.as_waveform()` 取得
所需样本。流式读取由 `Audio.stream()` 持续产生
[`AudioChunk`](/asr-data/api#audio-chunk)。

右侧 Example 按页面顺序连续执行，后面的示例会直接使用前面创建的变量。

## 音频来源 {id="source"}

`AudioSource` 是一个尚未解码的音频来源描述，而不是音频样本本身。创建来源时通常不
读取文件、不发起网络请求，也不执行解码；真正的 I/O 发生在 `probe()`、`open()`、
`load()` 或后续波形访问时。

> #### Example
>
> ```python
> from asr_data import AudioSource
>
> URL = "https://deepasset.oss-cn-beijing.aliyuncs.com/example.wav"
> source = AudioSource.from_url(URL)
>
> print(source.kind, source.url)
> ```

将“来源”和“解码结果”分开有三个目的：

- **延迟读取**：可以先组织任务，真正需要样本时再下载和解码。
- **统一入口**：文件、URL、内存字节和实时 PCM 最终都使用相同的加载与流式接口。
- **保留来源信息**：来源可以随业务数据传递，不需要长期保存体积更大的浮点样本。

支持五种来源：

| 创建方法        | 输入内容                 | 典型用途                         |
| --------------- | ------------------------ | -------------------------------- |
| `from_path()`   | 本地文件路径             | 本地数据集和缓存文件             |
| `from_url()`    | HTTP 或 HTTPS 地址       | 对象存储、数据服务和远程样本     |
| `from_bytes()`  | WAV、MP3 等编码音频字节  | 上传文件、消息体或内存缓存       |
| `from_base64()` | 编码音频的 base64 字符串 | JSON、日志或文本协议             |
| `from_pcm()`    | PCM S16LE 原始字节       | 麦克风、WebSocket 和实时音频设备 |

`from_bytes()` 中的字节仍然带有 WAV、MP3 等容器或编码信息，解码器可以从内容识别
音频格式。`from_pcm()` 没有文件头，因此必须显式提供 `sample_rate` 和
`channels`；每个样本使用有符号 16 位小端整数。

## 探测音频 {id="probe"}

`source.probe()` 读取音频容器或 PCM 参数，返回
[`AudioInfo`](/asr-data/api#audio-info)，但不会把采样解码为浮点数组。它适合在创建
标注文档前取得原始采样率、声道数、帧数和时长。

> #### Example
>
> ```python
> info = source.probe()
>
> print(info.sample_rate, info.channels)
> print(info.frame_count, info.duration_ms)
> print(info.source_format)
> ```

| 属性            | 含义                               |
| --------------- | ---------------------------------- |
| `sample_rate`   | 原始采样率                         |
| `channels`      | 原始声道数                         |
| `frame_count`   | 每个声道的有效采样帧数             |
| `duration_ms`   | 根据帧数和采样率计算的浮点毫秒时长 |
| `source_format` | 编码、采样率和声道组成的原始格式   |

异步程序使用 `await source.aprobe()`。URL 来源仍需要读取编码数据来探测容器，但
不会把整段音频转换为浮点样本。

## 加载音频 {id="load"}

`source.open()` 探测来源并创建尚未保存波形的 `Audio`；`source.load()` 创建同样
的对象，同时完成整段解码。异步程序分别使用 `await source.aopen()` 和
`await source.aload()`。

> #### Example
>
> ```python
> audio = source.open(id="example-001")
> waveform = audio.as_waveform().to_mono().resample(16_000)
>
> print(audio.info.duration_ms, audio.is_loaded)
> print(waveform.sample_rate, waveform.channels)
> print(waveform.samples.dtype, waveform.source_format)
> ```

第一次调用 `as_waveform()` 会在需要时解码并缓存完整波形，之后
`audio.is_loaded` 为 `True`。`audio.unload()` 只释放运行时样本，不会删除来源、
`info`、metadata、timeline 或标注。

| 对象 | 保存内容 | 典型操作 |
| --- | --- | --- |
| `Audio` | 来源、info、ID、metadata、timelines 和可选运行时波形 | 标注、流式读取、持久化 |
| `Waveform` | `float32` 样本、采样率、声道与原始格式 | 重采样、混声道、切片、播放 |

`Waveform` 提供以下基础信息：

| 属性            | 含义                                           |
| --------------- | ---------------------------------------------- |
| `samples`       | `float32` 样本；数值范围为 `[-1.0, 1.0]`       |
| `sample_rate`   | 每秒每个声道的采样帧数                         |
| `channels`      | 声道数                                         |
| `frame_count`   | 每个声道的采样帧数                             |
| `duration_ms`   | 根据帧数计算的时长，类型为 `float`             |
| `source_format` | 解码前检测到的编码、采样率和声道等原始格式信息 |

`duration_ms` 可能包含小数毫秒，这是采样帧数除以采样率的自然结果。只有创建
timeline 或 annotation 时，才需要按相应 API 的规则转换为毫秒整数。

## 音频处理 {id="processing"}

`Waveform` 的转换方法不会原地修改已有对象，而是返回新的 `Waveform`。这种设计可以保留原
始解码结果，并安全地从同一份音频产生不同采样率、声道或时间范围的版本。

> #### Example
>
> ```python
> processed = audio.as_waveform().to_mono().resample(16_000)
> clip = processed.slice_ms(0, 2_000)
> parts = processed.split_at_low_energy(1_000)
>
> print("processed:", processed.sample_rate, processed.channels)
> print("clip:", clip.duration_ms)
> print("parts:", [part.duration_ms for part in parts])
> ```

常用操作包括：

| 方法                                   | 作用                                      |
| -------------------------------------- | ----------------------------------------- |
| `resample(sample_rate)`                | 转换到目标采样率                          |
| `to_mono()`                            | 将所有声道混合为单声道                    |
| `channel(index)`                       | 提取一个声道，索引从 `0` 开始             |
| `slice_ms(start_ms, end_ms)`           | 按毫秒截取左闭右开的时间范围              |
| `split_at_low_energy(max_duration_ms)` | 将长音频优先拆在低能量位置                |
| `display(...)`                         | 在 IPython 或 Notebook 中播放指定时间范围 |

多声道样本按 frame 交错保存，`frame_count` 表示每个声道的帧数，而不是 `samples`
数组的元素总数。`slice_ms()` 和低能量切分会保留声道布局；需要单声道时应显式调用
`to_mono()`。

## 流式加载 {id="streaming"}

完整加载适合需要随机切片、全局分析或反复处理的音频；流式加载适合边接收边推理、控
制单次处理粒度或尽快得到首个结果的场景。

> #### Example
>
> ```python
> for chunk in audio.stream(chunk_size_ms=500):
>     model_input = chunk.as_waveform("mono").resample(16_000)
>     print(chunk.offset_ms, chunk.end_ms, chunk.is_final, model_input.frame_count)
> ```

`audio.stream(chunk_size_ms=...)` 返回同步迭代器，`audio.astream(...)` 返回异步
迭代器。`chunk_size_ms` 是目标分块时长；最后一块可能更短。同一个 `Audio` 同一时
刻只允许一个活动流。完整消费后，所有 chunk 会合并为该 `Audio` 的运行时波形并令
`is_loaded=True`；提前 `close()` 则保持未加载状态。

每个 `AudioChunk` 包含：

- `offset_ms`：本块相对整段音频起点的偏移。
- `end_ms`：本块在整段音频中的全局结束位置。
- `duration_ms`：本块的实际时长。
- `is_final`：是否为该来源的最后一块。
- `samples`、`sample_rate`、`channels` 和 `frame_count`：本块的音频数据与格式。

`AudioChunk.to_global_span()` 可以把模型返回的 chunk 内相对范围换算为整段音频的
全局范围；`chunk.timeline(channel)` 返回父 `Audio` 的完整 timeline，而不是只覆盖
当前 chunk 的临时时间轴。这样流式模型可以为过去的一段时间补写标注。
