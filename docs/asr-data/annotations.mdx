标注以 [`Audio`](/asr-data/api#audio) 为文档边界，以[`Timeline`](/asr-data/api#timeline) 表达一个声道上的时间范围，并把参考真值与模型预测分别保存。音频样本负责“听到什么”，标注结构负责“在什么时间发生了什么”。
右侧 Example 按页面顺序连续执行，后面的示例会直接使用前面创建的变量。

## 音频 {id="document"}

[`Audio`](/asr-data/api#audio) 是音频、时间轴与标注的共同所有者。它把原始
[`AudioSource`](/asr-data/api#audio-source)、`info`、稳定 ID、业务元数据、运行时
波形和多个声道 timeline 组织为一个对象。

> #### Example
>
> ```python
> from asr_data import AudioSource
>
> URL = "https://deepasset.oss-cn-beijing.aliyuncs.com/example.wav"
> audio = AudioSource.from_url(URL).open(id="example-001")
> audio.metadata["split"] = "train"
> print(audio.info.duration_ms, audio.timelines.keys())
> ```

`Audio` 可以在运行时缓存完整 [`Waveform`](/asr-data/api#waveform)，但数据库只保存
可重建状态，不保存大块浮点样本。因此从数据库读取的 `Audio.is_loaded` 为 `False`，
第一次访问波形视图或完整消费流时再加载样本。

| 内容         | 用途                                             |
| ------------ | ------------------------------------------------ |
| `source`     | 可重新加载或流式读取的原始音频来源               |
| `info` | 不解码样本即可使用的格式、帧数和时长             |
| `id`         | 数据库主键和跨系统关联标识                       |
| `metadata`   | split、语言、来源、业务标签等文档级信息          |
| `timelines`  | 按声道保存的时间轴及其 reference/prediction 标注 |

业务系统应尽量提供稳定且唯一的 `id`；省略时库会自动生成 ID。`metadata` 是普通字
典，不用于保存 annotation。

## 时间轴 {id="timeline"}

每个 [`Timeline`](/asr-data/api#timeline) 对应一个声道，并共享整条文档的
duration。 annotation 的所有时间范围都相对于所属 timeline 的起点。

> #### Example
>
> ```python
> timeline = audio.timeline("mono")
> print(timeline.duration_ms, audio.timelines.keys())
> ```

创建 `Audio` 时会调用 `source.probe()`，保存 `info`，并根据声道自动创建
时间轴。单声道使用 `mono`；双声道使用 `left`、`right`；更多声道继续使用 `2`、
`3` 等编号。时间轴时长由帧数计算并向上取整到整数毫秒。

`ensure_timeline()` 仍可用于取回已有时间轴或补充派生声道；新时间轴默认复用
`info` 确定的文档时长。

| 操作                       | 行为                                     |
| -------------------------- | ---------------------------------------- |
| `ensure_timeline(channel)` | 获取已有时间轴，不存在时创建             |
| `timeline(channel)`        | 只查询，不存在时返回 `None`              |
| `remove_timeline(channel)` | 删除声道及其全部标注，返回是否确实删除   |
| `timelines`                | 返回以标准化声道名称为键的全部时间轴字典 |

`timeline()` 不会隐式创建对象，这样可以避免拼错声道名称时意外产生空 timeline。声
道既可以使用名称，也可以使用整数；内部会转换为稳定的字符串键。

## 标注 {id="annotations"}

`TimeSpan` 由时间范围、置信度、来源和 `annotation` 组成。目前 annotation 可以是
AudioActivity、Token、Transcription 或 Speaker，分别表达声音事件、细粒度文本、
完整转写和说话人发话。

> #### Example
>
> ```python
> from asr_data.annotation import AudioActivity, Speaker, Transcription
>
> end_ms = timeline.duration_ms
> transcription = Transcription(
>     "甚至出现交易几乎停滞的情况", language="zh"
> )
> timeline.reference.annotate_span(
>     0, end_ms, AudioActivity(event="speech")
> )
> speaker_annotation = timeline.reference.annotate_span(
>     0, end_ms, Speaker("speaker-1", transcription=transcription)
> )
> prediction = timeline.prediction.annotate_span(
>     0, end_ms, transcription, source="qwen-asr"
> )
> ```

### 参考标注与预测标注 {id="groups"}

Timeline 将 annotation 分成两个集合，避免真值与模型结果混在一起：

| 集合                  | `source` 规则 | 典型内容                       |
| --------------------- | ------------- | ------------------------------ |
| `timeline.reference`  | 不需要来源    | 人工标注、审核结果、数据集真值 |
| `timeline.prediction` | 必须提供来源  | ASR、说话人识别或导入流程结果  |

Reference 的添加方法不接受 `source`。Prediction 必须提供非空字符串，例如
`"whisper"`、`"qwen-asr"` 或 `"import"`。不同类型的 prediction 可以来自不同模型
，例如 Speaker 使用 `"diarization-v2"`，Transcription 使用 `"qwen-asr"`。

### 标注类型 {id="adding"}

| 类型          | 添加方法              | Payload         | 用途                               |
| ------------- | --------------------- | --------------- | ---------------------------------- |
| AudioActivity | `annotate_span()`      | `AudioActivity` | 标记声音活动及可选的事件类别       |
| Transcription | `annotate_span()` | `Transcription` | 保存文本、语言、token 和置信度     |
| Speaker       | `annotate_span()`       | `Speaker`       | 保存说话人名称及可选转写           |

所有类型统一使用 `annotate_span()`。它返回
[`TimeSpan`](/asr-data/api#time-span)，可以继续读取 ID、范围、来源和
`annotation`，也可以直接调用 `span.as_waveform()`。

### 说话人、转写与 Token {id="payloads"}

[`Speaker`](/asr-data/api#speaker) 包含 `name` 和可选的 `transcription`。把转写
放在 `speaker.transcription` 中，可以直接表达某位说话人在该时间范围内的完整发话
。

[`Transcription`](/asr-data/api#transcription) 包含 `text`、`tokens`、`language`
和 `confidence`。更细粒度的 [`Token`](/asr-data/api#token) 位于
`Transcription.tokens` 中，可以携带文本、起止时间和置信度。

`TimeSpan` 不平铺 `.kind`、`.text`、`.name` 等字段，结构化内容统一通过
`.annotation` 访问。修改标注时应整体替换 annotation，例如把新的 `Speaker` 赋给
`speaker_annotation.annotation`。

```python
speaker_waveform = speaker_annotation.as_waveform()
timeline_waveform = timeline.as_waveform()
```

### 完整校验规范 {id="validation"}

下面的规则是 annotation 的完整有效性标准。重叠校验以单个声道的
[`Timeline`](/asr-data/api#timeline) 为边界：不同声道、reference 与 prediction
彼此独立，可以互相重叠。

#### 当前状态与删除

TimeSpan 不再具有 `status`。集合中的每条 annotation 都表示当前有效状态：修改
时直接替换内容，删除时从集合中移除，不使用 `partial`、`revised`、`deleted` 等逻
辑状态。

#### 时间范围

- TimeSpan 范围必须满足
  `0 <= start_ms <= end_ms <= timeline.duration_ms`。
- 范围采用半开区间 `[start_ms, end_ms)`。`[0, 100)` 与 `[100, 200)` 首尾相接，
  不构成重叠。
- `start_ms == end_ms` 的零长度范围合法，并且不会与其他范围重叠。
- Python `annotate_span()` 会立即检查起止顺序和 timeline 边界。Rust 直接构造
  `TimeRange` 时应遵守同一标准；`Audio::validate()` 会检查 annotation 是否超
  出 timeline 结束位置。
- Python 中 Token 的 `start_ms` 和 `end_ms` 必须同时提供或同时省略。带范围的
  Token 必须完全位于所属 Transcription 或 Speaker annotation 内；添加或整体替换
  payload 时都会检查。

#### Reference 与 Prediction 来源

- Reference annotation 不携带 source。Python reference API 不接收 source；Rust
  `push_reference()` 会将 source 设为 `None`，整体验证会拒绝通过直接修改得到的
  非空 reference source。
- 每条 Prediction annotation 都必须有 source，而且 source 至少包含一个非空白字
  符。`""` 和 `"   "` 无效。
- source 使用原始字符串精确比较，区分大小写，不会自动 trim 或标准化。因此
  `"asr"`、`"ASR"` 和 `" asr "` 是三个不同来源。

#### 重叠规则

下表适用于 reference，也适用于 prediction 的同一个 source。表中的“允许”只表示
没有新增的重叠限制，其他范围、source 和 payload 规则仍然有效。

| 组合 | 是否允许重叠 | 分组与说明 |
| ---- | ------------ | ---------- |
| 同 event 的 AudioActivity ↔ AudioActivity | 不允许 | 相同事件类别共用一条区间轨道；两个 `event=None` 也属于同一组。 |
| 不同 event 的 AudioActivity ↔ AudioActivity | 允许 | 例如语音与背景音乐可以同时发生。 |
| AudioActivity ↔ Speaker | 允许 | 声音活动可以覆盖说话人区间。 |
| AudioActivity ↔ Transcription | 允许 | 声音活动可以覆盖转写区间。 |
| Speaker ↔ Speaker | 同名不允许，不同名允许 | `name` 使用大小写敏感的精确匹配。 |
| 顶层 Transcription ↔ 顶层 Transcription | 不允许 | 重叠内容应写入同一条 Transcription。 |
| 顶层 Transcription ↔ 带 transcription 的 Speaker | 不允许 | 避免同一时间范围出现两份转写。 |
| 顶层 Transcription ↔ 不带 transcription 的 Speaker | 允许 | Speaker 只表达说话人身份。 |
| 不同 name 且都带 transcription 的 Speaker | 允许 | 用于表达不同说话人的同时说话。 |

`Token`、`Sentence` 和 `Language` 目前没有新增的集合级重叠规则。同一个
prediction source 使用上表；不同 source 可以自由重叠。例如两个声音活动检测模型
可以保存彼此重叠的结果。

#### 去重、冲突与原子性

- 内容完全相同的重复插入具有幂等性：返回已有 annotation，不插入第二条。比较会
  忽略自动生成的 `id`，但会比较 range、source、confidence 和完整 payload。
- 检测到冲突时会返回错误，不会自动合并、裁剪、拆分、覆盖或移动范围。
- Python 只能用相同 annotation 类型整体替换当前内容。替换会在候选副本上
  完成全部校验；失败时原 annotation 保持不变。
- `prediction.relabel_source()` 会先校验改名后的完整 prediction 集合。只要一条冲
  突，整个重命名失败且不修改任何 annotation。

#### 校验发生的位置

| 入口 | 校验行为 |
| ---- | -------- |
| Python `reference/prediction.annotate_span()` | 添加前检查范围、source、annotation 和重叠规则。 |
| Rust `Timeline::push_reference/push_prediction()` | 幂等插入，并检查 source 与重叠规则。 |
| Python `span.annotation = ...` | 使用候选副本进行原子校验。 |
| Python/Rust source 重命名 | 校验完整候选集合，成功后一次性提交。 |
| `Audio.validate()` | 重新检查声道、timeline、annotation 边界、source 和集合重叠。 |
| 数据库读取、插入和更新 | 使用 schema v10，并通过 `Audio.validate()` 拒绝无效文档。 |

Rust 的 `reference` 和 `prediction` 是公开 `Vec`，可以绕过 `push_*()` 直接修改；这
种用法必须在持久化前调用 `Audio::validate()`。数据库写入也会自动执行同样的整
体验证。

### 评测 {id="evaluation"}

[`Timeline.eval()`](/asr-data/api#timeline-eval) 比较 reference 与 prediction。
不传参数时会自动发现所有可评测 source；也可以分别通过 `transcription` 和
`activity` 限定一个或多个来源。

> #### Example
>
> ```python
> timeline.prediction.annotate_span(
>     0, end_ms, AudioActivity(event="speech"), source="silero-vad"
> )
> evaluation = timeline.eval()
>
> print(evaluation.transcription["qwen-asr"].cer)
> print(evaluation.transcription["qwen-asr"].normalized_reference)
> print(evaluation.activity["silero-vad"].f1)
> print(evaluation.activity["silero-vad"].events["speech"].f1)
> ```

自动发现转写来源时会检查 Transcription、Sentence，以及带 transcription 的
Speaker；Activity 来源只来自 AudioActivity prediction。显式参数接受字符串或字符
串列表：

```python
selected = timeline.eval(
    transcription=["qwen-asr", "whisper"],
    activity="silero-vad",
)
```

只传入其中一个参数时只评测该任务。显式来源不存在时抛出错误；完全没有可评测的
reference 和 prediction 时同样报错。

#### 转写评测

转写评测先按时间顺序组合 Transcription，以及 Speaker 中携带的
transcription，再计算字符错误率。默认启用内嵌中文 TN，将书写形式标准化为口语形式
后去除空白和标点。例如 `"2024年"` 与 `"二零二四年"` 会被视为相同文本。FST 资源已
经编译进 asr-data，不需要安装或调用外部标准化服务。

| 字段                    | 含义                         |
| ----------------------- | ---------------------------- |
| `cer`                   | 字符错误率 `(S + D + I) / N` |
| `matches`               | 匹配字符数                   |
| `substitutions`         | 替换字符数                   |
| `deletions`             | 删除字符数                   |
| `insertions`            | 插入字符数                   |
| `precision`             | 字符级准确率                 |
| `recall`                | 字符级召回率                 |
| `f1`                    | 字符级 F1                    |
| `exact_match`           | 标准化后的文本是否完全相同   |
| `reference`             | 原始参考文本                 |
| `hypothesis`            | 原始预测文本                 |
| `normalized_reference`  | 标准化后的参考文本           |
| `normalized_hypothesis` | 标准化后的预测文本           |
| `normalization`         | `"zh_tn"` 或 `"none"`        |

如需严格按原始字符评测，可以关闭标准化：

```python
raw = timeline.eval(transcription="qwen-asr", normalize=False)
print(raw.transcription["qwen-asr"].cer)
```

#### 声音活动评测

Activity 评测只使用 `annotate_span()` 创建的标注，不会根据 Speaker 或
Transcription 自动推断活动区间。reference 至少有一条 AudioActivity 才表示该
timeline 已完成活动标注；没有 reference Activity 的 timeline 不参与该任务。已标
注 timeline 中，Activity 区间之外的部分视为无活动。

评测分为两层。`evaluation.activity[source]` 会忽略 `event`，把所有活动区间合并后
评测“是否存在声音活动”；`evaluation.activity[source].events[event]` 再按事件类别
做 one-vs-rest 评测。`event=None` 表示只知道该区间存在活动、但不知道类别，因此参
与整体活动评测，并在事件级评测中遮罩该区间，既不算对也不算错。

| 字段                | 含义                               |
| ------------------- | ---------------------------------- |
| `reference_ms`      | reference 中的活动总时长           |
| `predicted_ms`      | prediction 中的活动总时长          |
| `true_positive_ms`  | 正确预测为活动的时长               |
| `true_negative_ms`  | 正确预测为无活动的时长             |
| `false_positive_ms` | 把无活动误判为活动的时长           |
| `false_negative_ms` | 漏掉的活动时长                     |
| `precision`         | 活动区间准确率                     |
| `recall`            | 活动区间召回率                     |
| `f1`                | 活动区间 F1                        |
| `iou`               | reference 与 prediction 的区间 IoU |
| `events`            | 以 event 名称为键的事件级结果      |

`TimelineEvaluation.transcription` 和 `activity` 都是以 source 为键的字典；未评测
的任务返回空字典。

#### 数据集评测

[`evaluate_dataset()`](/asr-data/api#evaluate-dataset) 接受一组 Audio，
[`AudioDB.eval()`](/asr-data/api#audio-db-eval) 则自动分页评测数据库中所有匹配记
录：

```python
from asr_data import evaluate_dataset

memory_result = evaluate_dataset(docs, normalize=False)
database_result = db.eval(
    transcription=["qwen-asr", "whisper"],
    activity="silero-vad",
    metadata={"split": "test"},
)

print(database_result.transcription["qwen-asr"].cer)
print(database_result.transcription["qwen-asr"].coverage)
print(database_result.activity["silero-vad"].f1)
print(database_result.activity["silero-vad"].events["speech"].f1)
```

数据集 CER 累加每条 timeline 的替换、删除、插入和参考字符数后计算，不会平均单条
CER，也不会把不同文档的文本拼接后对齐。Activity 的整体指标与各 event 指标都先累
加各 timeline 的 TP、TN、FP、FN 毫秒数，再计算总体 precision、recall、F1 和 IoU。

每个 source 都记录 `evaluated_timelines`、`unannotated_timelines`、
`missing_predictions`、`coverage` 和对应 ID。这样模型只覆盖部分数据时不会被误认
为完整数据集指标。

## 便捷方法 {id="helpers"}

Timeline 提供组合转写、按来源查询和批量维护等便捷方法，不需要使用者手动遍历全部
annotation。

> #### Example
>
> ```python
> reference_text = timeline.reference.transcript()
> predicted_text = timeline.prediction.transcript("qwen-asr")
> items = timeline.prediction.by_source("qwen-asr")
>
> print(reference_text.text)
> print(predicted_text.text)
> print(timeline.prediction.sources["transcription"], len(items))
> audio.validate()
> ```

| 方法                                  | 作用                             |
| ------------------------------------- | -------------------------------- |
| `reference.transcript()`              | 组合 reference 中的转写          |
| `prediction.transcript(source)`       | 只组合指定来源的转写             |
| `prediction.by_source(source)`        | 返回指定来源的全部 annotation    |
| `prediction.sources`                  | 按 annotation 类型返回来源字典   |
| `prediction.relabel_source(old, new)` | 批量重命名来源                   |
| `prediction.remove_by_source(source)` | 删除指定来源的全部 prediction    |
| `reference.spans`                     | 返回全部 reference TimeSpan      |
| `prediction.spans`                    | 返回全部 prediction TimeSpan     |
| `audio.validate()`                    | 检查音频、时间轴、范围和来源约束 |

`transcript()` 会把 Transcription annotation 和带 transcription 的
Speaker annotation 按开始时间组合为
[`Transcript`](/asr-data/api#transcript)。AudioActivity 没有文字，不参与组合。

Prediction 的 `transcript()` 必须指定 source，防止混合不同模型的输出。
`relabel_source()` 适合统一历史模型名称，`remove_by_source()` 适合清理一次错误推
理产生的全部结果。写入数据库前可以调用 `audio.validate()` 做一次完整校验。
