标注以 [`AudioDoc`](/asr-data/api#audio-doc) 为文档边界，以
[`Timeline`](/asr-data/api#timeline) 表达一个声道上的时间范围，并把参考真值与模
型预测分别保存。音频样本负责“听到什么”，标注结构负责“在什么时间发生了什么”。

右侧 Example 按页面顺序连续执行，后面的示例会直接使用前面创建的变量。

## 文档 {id="document"}

[`AudioDoc`](/asr-data/api#audio-doc) 把原始
[`AudioSource`](/asr-data/api#audio-source)、稳定 ID、业务元数据和多个声道
timeline 组织为一条可标注、可持久化的记录。

> #### Example
>
> ```python
> from asr_data import AudioDoc, AudioSource
>
> URL = "https://deepasset.oss-cn-beijing.aliyuncs.com/example.wav"
> source = AudioSource.from_url(URL)
> audio = source.load()
> doc = AudioDoc(source, id="example-001")
> doc.metadata["split"] = "train"
> ```

文档保存的是来源描述，而不是解码后的 [`Audio`](/asr-data/api#audio) 或
[`AudioChunk`](/asr-data/api#audio-chunk) 样本。因此同一条记录可以按需重新加载、
重采样或流式读取，同时避免在数据库中重复保存大块浮点数据。

| 内容        | 用途                                             |
| ----------- | ------------------------------------------------ |
| `source`    | 可重新加载或流式读取的原始音频来源               |
| `id`        | 数据库主键和跨系统关联标识                       |
| `metadata`  | split、语言、来源、业务标签等文档级信息          |
| `timelines` | 按声道保存的时间轴及其 reference/prediction 标注 |

业务系统应尽量提供稳定且唯一的 `id`；省略时库会自动生成 ID。`metadata` 是普通字
典，不用于保存 annotation。

## 时间轴 {id="timeline"}

每个 [`Timeline`](/asr-data/api#timeline) 对应一个声道，并共享整条文档的
duration。 annotation 的所有时间范围都相对于所属 timeline 的起点。

> #### Example
>
> ```python
> timeline = doc.ensure_timeline("mono", duration_ms=audio.duration_ms)
> print(timeline.duration_ms, doc.timelines.keys())
> ```

`ensure_timeline()` 在声道不存在时创建它，存在时返回已有对象。创建第一个
timeline 必须提供 duration；它接受整数或浮点毫秒，浮点值向上取整。后续声道共享相
同 duration，可以省略。

| 操作                       | 行为                                     |
| -------------------------- | ---------------------------------------- |
| `ensure_timeline(channel)` | 获取已有时间轴，不存在时创建             |
| `timeline(channel)`        | 只查询，不存在时返回 `None`              |
| `remove_timeline(channel)` | 删除声道及其全部标注，返回是否确实删除   |
| `timelines`                | 返回以标准化声道名称为键的全部时间轴字典 |

`timeline()` 不会隐式创建对象，这样可以避免拼错声道名称时意外产生空 timeline。声
道既可以使用名称，也可以使用整数；内部会转换为稳定的字符串键。

## 标注 {id="annotations"}

annotation 由时间范围、状态、置信度、来源和 payload 组成。目前可以添加 Speech、
Transcription 和 Speaker，分别表达“是否有人声”“说了什么”和“谁在说”。

> #### Example
>
> ```python
> from asr_data.annotation import Speaker, Transcription
>
> end_ms = timeline.duration_ms
> transcription = Transcription(
>     "甚至出现交易几乎停滞的情况", language="zh"
> )
> timeline.reference.add_speech(0, end_ms)
> speaker_annotation = timeline.reference.add_speaker(
>     0, end_ms, Speaker("speaker-1", transcription=transcription)
> )
> prediction = timeline.prediction.add_transcription(
>     0, end_ms, transcription, source="qwen-asr"
> )
> ```

### 参考标注与预测标注 {id="groups"}

Timeline 将 annotation 分成两个集合，避免真值与模型结果混在一起：

| 集合                  | `source` 规则 | 典型内容                       |
| --------------------- | ------------- | ------------------------------ |
| `timeline.reference`  | 不需要来源    | 人工标注、审核结果、数据集真值 |
| `timeline.prediction` | 必须提供来源  | ASR、说话人识别或导入流程结果  |

Reference 的添加方法不接受 `source`。Prediction 必须提供非空字符串，例如
`"whisper"`、`"qwen-asr"` 或 `"import"`。不同类型的 prediction 可以来自不同模型
，例如 Speaker 使用 `"diarization-v2"`，Transcription 使用 `"qwen-asr"`。

### 标注类型 {id="adding"}

| 类型          | 添加方法              | Payload         | 用途                           |
| ------------- | --------------------- | --------------- | ------------------------------ |
| Speech        | `add_speech()`        | 无              | 标记某个时间范围内存在人声     |
| Transcription | `add_transcription()` | `Transcription` | 保存文本、语言、token 和置信度 |
| Speaker       | `add_speaker()`       | `Speaker`       | 保存说话人名称及可选转写       |

Silence 类型及 `add_silence()` 已删除。添加方法返回
[`Annotation`](/asr-data/api#annotation)，可以继续读取 ID、范围、状态、来源和
payload。

### 说话人、转写与 Token {id="payloads"}

[`Speaker`](/asr-data/api#speaker) 包含 `name` 和可选的 `transcription`。把转写
放在 `speaker.transcription` 中，可以直接表达某位说话人在该时间范围内的完整发话
。

[`Transcription`](/asr-data/api#transcription) 包含 `text`、`tokens`、`language`
和 `confidence`。更细粒度的 [`Token`](/asr-data/api#token) 位于
`Transcription.tokens` 中，可以携带文本、起止时间和置信度。

`Annotation` 不再平铺 `.text`、`.name` 等字段，结构化内容统一通过 `.payload` 访
问。修改 annotation 时应整体替换 payload，例如把新的 `Speaker` 赋给
`speaker_annotation.payload`。

### 状态与来源 {id="provenance"}

[`AnnotationStatus`](/asr-data/api#annotation-status) 描述标注在处理流程中的状态
：

| 状态      | 含义                       |
| --------- | -------------------------- |
| `partial` | 尚未稳定的流式或中间结果   |
| `final`   | 最终结果，会参与组合转写   |
| `revised` | 已经人工或程序修订的结果   |
| `deleted` | 保留记录但在语义上逻辑删除 |

Reference 不携带来源；Prediction 的 `source` 直接使用模型或流程名称。旧的
`AnnotationSourceKind` 以及 `user/model/stage/system` 分类已经删除。

### 范围与去重 {id="validation"}

所有范围必须满足 `0 <= start_ms <= end_ms <= timeline.duration_ms`。Token 的
`start_ms` 和 `end_ms` 必须同时提供或同时省略；带时间范围的 token 必须完全位于所
属 Speaker 或 Transcription annotation 内。

内容完全相同的重复 `add_*()` 调用会返回已有 annotation，不会插入第二条记录。去重
比较业务内容，自动生成的 annotation ID 不参与比较。

## 便捷方法 {id="helpers"}

Timeline 提供组合转写、按来源查询和批量维护等便捷方法，不需要使用者手动遍历全部
annotation。

> #### Example
>
> ```python
> reference_text = timeline.reference.transcript()
> predicted_text = timeline.prediction.transcript("qwen-asr")
> items = timeline.prediction.by_source("qwen-asr")
>
> print(reference_text.text)
> print(predicted_text.text)
> print(timeline.prediction.sources, len(items))
> doc.validate()
> ```

| 方法                                  | 作用                             |
| ------------------------------------- | -------------------------------- |
| `reference.transcript()`              | 组合 reference 中的最终转写      |
| `prediction.transcript(source)`       | 只组合指定来源的最终转写         |
| `prediction.by_source(source)`        | 返回指定来源的全部 annotation    |
| `prediction.sources`                  | 返回当前所有 prediction 来源     |
| `prediction.relabel_source(old, new)` | 批量重命名来源                   |
| `prediction.remove_by_source(source)` | 删除指定来源的全部 prediction    |
| `reference.annotations`               | 返回全部 reference annotation    |
| `prediction.annotations`              | 返回全部 prediction annotation   |
| `doc.validate()`                      | 检查文档、时间轴、范围和来源约束 |

`transcript()` 会把最终状态的 Transcription annotation 和带 transcription 的
Speaker annotation 按开始时间组合为
[`Transcript`](/asr-data/api#transcript)。Speech 没有文字，不参与组合；`partial`
和 `deleted` 等非最终状态也不会被当作最终文本。

Prediction 的 `transcript()` 必须指定 source，防止混合不同模型的输出。
`relabel_source()` 适合统一历史模型名称，`remove_by_source()` 适合清理一次错误推
理产生的全部结果。写入数据库前可以调用 `doc.validate()` 做一次完整校验。
