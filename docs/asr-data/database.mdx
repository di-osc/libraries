`AudioDB` 将音频来源、时间轴、标注和业务元数据保存在一个 SQLite 文件中。
推荐使用 `.sqlite` 后缀。

## 打开数据库 {id="open"}

```python
from asr_data import AudioDB

db = AudioDB("dataset.sqlite")
```

不存在的文件会自动创建。数据库带有专用的 SQLite application ID 和 schema
version，打开时会校验文件格式。

## 增删改查 {id="crud"}

```python
# 新增；ID 已存在时会报错
db.insert(audio)

# 按 ID 获取；不存在时抛出 KeyError
loaded = db["call-001"]

# 判断和计数
if "call-001" in db:
    print(len(db))

# 更新已存在的记录
loaded.metadata["checked"] = True
changed = db.update(loaded)

# 删除
deleted = db.delete("call-001")
```

`update()` 只写入发生变化的部分；没有变化时返回 `False`。批量更新使用单个
SQLite 事务：

```python
updated_count = db.update_many(audios)
```

直接迭代数据库会通过内部游标懒加载记录：

```python
for audio in db:
    print(audio.id)
```

## 筛选和分页 {id="query"}

查询结果按 `audio_id` 排序。使用上一页最后一条记录的 ID 作为 `after` 游标：

```python
first_page = db.query(
    limit=100,
    metadata={"split": "train"},
    min_duration_ms=1_000,
    max_duration_ms=30_000,
)

second_page = db.query(
    limit=100,
    after=first_page[-1].id,
    metadata={"split": "train"},
    min_duration_ms=1_000,
    max_duration_ms=30_000,
)
```

元数据过滤要求存储值与查询值匹配，适合数据集切分、语言、来源等结构化字段。

## 数据库元数据 {id="metadata"}

数据库级元数据不属于单条音频，适合记录数据集版本、标注运行和模型配置：

```python
db.set_metadata(
    "annotation_runs",
    {"qwen3-asr": {"language": "zh", "version": "1"}},
)

run = db.metadata_value("annotation_runs")
all_metadata = db.metadata
db.delete_metadata("annotation_runs")
```

## 只读模式 {id="readonly"}

处理不应修改的评测集或生产快照时，以只读模式打开：

```python
db = AudioDB("evaluation.sqlite", read_only=True)
```

只读模式要求数据库文件已经存在并且格式有效。
