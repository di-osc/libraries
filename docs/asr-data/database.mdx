[`AudioDB`](/asr-data/api#audio-db) 使用一个 SQLite 文件持久化
[`AudioDoc`](/asr-data/api#audio-doc)。数据库保存音频来源、`AudioInfo`、
timeline、annotation 和业务元数据，不复制解码后的浮点样本。

右侧 Example 按页面顺序连续执行，后面的示例会直接使用前面创建的数据库和文档。

## 打开数据库 {id="open"}

使用 `AudioDB.create(path)` 创建新数据库，使用 `AudioDB.open(path)` 打开已有
数据库。两个入口不会互相兜底，因此路径写错时不会意外创建空数据库。

> #### Example
>
> ```python
> from tempfile import TemporaryDirectory
>
> from asr_data import AudioDB
>
> directory = TemporaryDirectory()
> path = f"{directory.name}/dataset.db"
> db = AudioDB.create(path)
> opened = AudioDB.open(path)
> ```

`create()` 在文件已经存在时抛出 `FileExistsError`；`open()` 在文件不存在时抛出
`FileNotFoundError`。推荐使用 `.db` 后缀，以便工具和使用者识别文件类型。数据库包含专用 SQLite
application ID 和 schema version。当前仅接受 schema v8；打开普通 SQLite 文件或其
他版本时会立即报错，而不是尝试迁移或按错误结构读写。

一个数据库可以包含大量 [`AudioDoc`](/asr-data/api#audio-doc)，但每个文档 ID 必须
唯一。路径只决定数据库文件位置，不会改变
[`AudioSource`](/asr-data/api#audio-source) 中保存的本地路径或 URL。

`AudioInfo` 与文档一起持久化。因此读取数据库记录时不会重新打开本地文件或请求
URL；即使原始来源暂时不可用，仍然可以读取采样率、声道、帧数、时长和全部标注。

## 增删改查 {id="crud"}

[`AudioDB`](/asr-data/api#audio-db) 同时提供显式方法和类似字典的读取协议。插入、
更新和删除都以 [`AudioDoc`](/asr-data/api#audio-doc) ID 为记录主键。

> #### Example
>
> ```python
> from asr_data import AudioDoc, AudioSource
>
> URL = "https://deepasset.oss-cn-beijing.aliyuncs.com/example.wav"
> doc = AudioDoc(AudioSource.from_url(URL), id="call-001")
> doc.metadata["split"] = "train"
>
> # 增
> db.insert(doc)
> # 查
> loaded = db["call-001"]
> print(loaded.id, loaded.metadata)
> # 改
> loaded.metadata["checked"] = True
> changed = db.update(loaded)
> # 删
> deleted = db.delete("call-001")
>
> print(changed, deleted)
> db.insert(loaded)  # 供后续示例继续使用
> ```

| 操作                | 行为                                           |
| ------------------- | ---------------------------------------------- |
| `insert(doc)`       | 新增文档；ID 已存在时返回错误                  |
| `db[audio_id]`      | 按 ID 读取；不存在时抛出 `KeyError`            |
| `update(doc)`       | 只写入发生变化的部分；没有变化时返回 `False`   |
| `update_many(docs)` | 在单个 SQLite 事务中批量更新，返回实际更新数量 |
| `delete(audio_id)`  | 删除指定文档，返回是否确实删除                 |
| `audio_id in db`    | 判断 ID 是否存在                               |
| `len(db)`           | 返回文档数量                                   |
| `for doc in db`     | 通过内部游标懒加载全部文档                     |

`update()` 不执行隐式插入：目标 ID 必须已经存在。批量更新使用一个事务，任一写入
失败时不会留下只更新了一部分的数据。

数据库会为每条记录自动维护创建和修改时间。`insert()` 使用同一时刻初始化两个时间
；只有内容确实发生变化、`update()` 返回 `True` 时才更新修改时间。无变化的更新不
会改变时间，`update_many()` 中实际变化的记录使用同一个批次时间。

这两个时间属于数据库存储信息，不会写入 [`AudioDoc`](/asr-data/api#audio-doc) 或
`doc.metadata`，用户也不能手动设置。

### 查询与分页 {id="query"}

`query()` 可以同时按 metadata、时长、创建时间和修改时间筛选，并使用稳定的
AudioDoc ID 游标分页。时间参数只接受带时区的 Python `datetime`，内部统一转换为
UTC Unix 毫秒。

> #### Example
>
> ```python
> from datetime import datetime, timedelta, timezone
>
> now = datetime.now(timezone.utc)
> page = db.query(
>     limit=10,
>     created_from=now - timedelta(days=7),
>     created_until=now,
>     updated_from=now - timedelta(hours=24),
>     metadata={"split": "train"},
> )
> print([item.id for item in page])
> ```

时间范围统一使用半开区间：

```text
from <= timestamp < until
```

`created_from`、`created_until` 筛选创建时间，`updated_from`、`updated_until` 筛
选最后一次实际修改时间。四个条件都可以单独使用；传入无时区 `datetime` 或反向范围
会抛出 `ValueError`。

查询结果按 `audio_id` 排序。读取下一页时，将上一页最后一条记录的 ID 传给
`after`：

```python
next_page = db.query(limit=10, after=page[-1].id)
```

`after` 是游标而不是数字页码，因此即使数据库较大，也不需要跳过前面所有记录。
metadata 过滤要求存储值与查询值匹配，适合数据集 split、语言、来源和审核状态等结
构化字段。

## 数据集评测 {id="evaluation"}

`db.eval()` 使用与 `query()` 相同的筛选条件，并通过 AudioDoc ID 游标自动读取全部
匹配记录。`batch_size` 只控制每次从 SQLite 读取的数量，不限制最终评测集大小。

> #### Example
>
> ```python
> result = db.eval(
>     transcription=["qwen-asr", "whisper"],
>     speech="silero-vad",
>     metadata={"split": "test"},
>     batch_size=100,
> )
>
> qwen = result.transcription["qwen-asr"]
> print(qwen.cer, qwen.coverage)
> print(result.speech["silero-vad"].f1)
> ```

不传 `transcription` 和 `speech` 时自动发现全部可评测来源。结果按 source 分组，并
记录缺少 reference、缺少 prediction 和实际参与聚合的 timeline 数量。只读数据库同
样可以执行评测。

## 数据库元数据 {id="metadata"}

数据库级 metadata 描述整个数据集，不属于任何一条 AudioDoc，适合保存数据集版本、
标注批次或模型配置。

> #### Example
>
> ```python
> db.set_metadata("dataset_version", "2026-07")
> print(db.metadata_value("dataset_version"))
> print(db.metadata)
> ```

`set_metadata(key, value)` 新增或覆盖一个值，`metadata_value(key)` 查询单个值，
`delete_metadata(key)` 删除并返回是否存在。`db.metadata` 返回当前全部数据库级元
数据。

业务上只属于单条音频的信息应放在 `doc.metadata`；只有需要对整个数据库共享的信息
才应放在 `db.metadata`，这样查询和迁移时两者的作用域保持清晰。

## 只读模式 {id="readonly"}

评测集、发布快照或其他不应被修改的数据库，可以使用 `read_only=True` 打开。

> #### Example
>
> ```python
> read_only = AudioDB.open(path, read_only=True)
> print(read_only["call-001"].id)
> ```

只读模式要求文件已经存在且格式有效，读取、查询和迭代仍然可用；插入、更新、删除以
及数据库 metadata 写操作会失败。只读连接不会自动创建缺失文件。

需要删除记录时，应使用可写连接调用 `db.delete(audio_id)`。临时示例中的目录会在
Python 进程结束时自动清理；生产数据库应使用稳定的绝对路径并纳入备份策略。
