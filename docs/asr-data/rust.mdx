## AudioSource {id="audio-source",tag="enum"}

描述尚未解码的音频来源。它是一个 enum，因此来源类型在编译期明确，不依赖字符串
kind。`load()` 解码为完整 [`Audio`](#audio)，`stream()` 创建逐块解码的
[`AudioStream`](#audio-stream)。

| Variant                                     | Description           |
| ------------------------------------------- | --------------------- |
| `Path(PathBuf)`                             | 本地音频文件。        |
| `Url(String)`                               | HTTP 或 HTTPS 地址。  |
| `EncodedBytes(Vec<u8>)`                     | WAV、MP3 等编码字节。 |
| `Base64(String)`                            | base64 编码的音频。   |
| `PcmS16Le { bytes, sample_rate, channels }` | PCM S16LE 音频。      |

## AudioSource::from_url {id="audio-source-from-url",tag="associated function"}

从 URL 创建音频来源。构造过程不发起请求，调用 `load()` 时才下载并解码。

> #### Example
>
> ```rust
> use asr_data::AudioSource;
>
> const URL: &str =
>     "https://deepasset.oss-cn-beijing.aliyuncs.com/example.wav";
>
> fn main() -> Result<(), Box<dyn std::error::Error>> {
>     let source = AudioSource::from_url(URL);
>     let audio = source.load()?;
>     println!("{:.0} ms", audio.info.duration_ms());
>     Ok(())
> }
> ```

| Name        | Description                                           |
| ----------- | ----------------------------------------------------- |
| `url`       | 可转换为 `String` 的 URL。~~impl Into&lt;String&gt;~~ |
| **RETURNS** | 尚未加载的来源。~~AudioSource~~                       |

## AudioSource constructors {id="audio-source-constructors",tag="associated functions"}

其他构造方法直接对应 enum 的其余来源形式。

| Method                                         | Description                |
| ---------------------------------------------- | -------------------------- |
| `from_path(path)`                              | 从本地路径创建来源。       |
| `from_encoded_bytes(bytes)`                    | 从编码字节创建来源。       |
| `from_base64(data)`                            | 从 base64 字符串创建来源。 |
| `from_pcm_s16le(bytes, sample_rate, channels)` | 从 PCM S16LE 创建来源。    |
| `new(path_or_url)`                             | 根据是否包含 `://` 推断。  |

优先使用显式的 `from_*()`，避免让调用者猜测 `new()` 的路径/URL 判定规则。

## AudioSource::load / stream {id="audio-source-load",tag="methods"}

`load()` 完整解码来源并返回 [`Audio`](#audio)；`stream(chunk_size_ms)` 只探测
来源，返回随迭代增长的 [`AudioStream`](#audio-stream)。

> #### Example
>
> ```rust
> use asr_data::AudioSource;
>
> const URL: &str =
>     "https://deepasset.oss-cn-beijing.aliyuncs.com/example.wav";
>
> fn main() -> Result<(), Box<dyn std::error::Error>> {
>     let source = AudioSource::from_url(URL);
>     let mut audio = source.load_with_id("example-001")?;
>     let waveform = audio.as_waveform()?.to_mono()?.resample(16_000)?;
>     println!("{} Hz, {} channel", waveform.sample_rate, waveform.channels);
>     Ok(())
> }
> ```

| Method                                  | Returns                               | Description              |
| --------------------------------------- | ------------------------------------- | ------------------------ |
| `load()`                                | `anyhow::Result<Audio>`               | 完整加载并自动生成 ID。  |
| `load_with_id(id)`                      | `anyhow::Result<Audio>`               | 完整加载并指定文档 ID。  |
| `stream(chunk_size_ms)`                 | `anyhow::Result<AudioStream>`         | 创建流并自动生成 ID。    |
| `stream_with_id(id, chunk_size_ms)`     | `anyhow::Result<AudioStream>`         | 创建流并指定文档 ID。    |

异步完整加载使用 `source.aload().await`。Rust 的 `AudioStream` 实现标准
`Iterator`；需要异步调度时可以在调用侧放入阻塞任务。

## AudioSource::probe {id="audio-source-probe",tag="method"}

读取来源的格式和时长信息，但不解码采样数据。异步环境使用
`source.aprobe().await`。

> #### Example
>
> ```rust
> use asr_data::AudioSource;
>
> const URL: &str =
>     "https://deepasset.oss-cn-beijing.aliyuncs.com/example.wav";
>
> fn main() -> anyhow::Result<()> {
>     let info = AudioSource::from_url(URL).probe()?;
>     println!("{} Hz, {} ch, {:.2} ms",
>         info.sample_rate, info.channels, info.duration_ms());
>     Ok(())
> }
> ```

| Name        | Description                                  |
| ----------- | -------------------------------------------- |
| `&self`     | 借用来源。                                   |
| **RETURNS** | 音频元信息。~~anyhow::Result&lt;AudioInfo&gt;~~ |

## Waveform transformations {id="waveform-transformations",tag="methods"}

变换方法借用 `&self` 并返回新的 `Waveform`，不会原地修改样本。

> #### Example
>
> ```rust
> use asr_data::{Waveform, DurationMs};
>
> const URL: &str =
>     "https://deepasset.oss-cn-beijing.aliyuncs.com/example.wav";
>
> fn main() -> Result<(), Box<dyn std::error::Error>> {
>     let audio = Waveform::from_url(URL)?;
>     let mono = audio.to_mono()?;
>     let resampled = mono.resample(16_000)?;
>     let clip = resampled.slice_ms(0, 1_000);
>     let parts = resampled.split_at_low_energy(DurationMs(10_000))?;
>
>     println!("{} samples", clip.samples.len());
>     println!("{} parts", parts.len());
>     Ok(())
> }
> ```

| Method                              | Returns                          | Description           |
| ----------------------------------- | -------------------------------- | --------------------- |
| `to_mono()`                         | `Result<Waveform, AudioError>`      | 混合为单声道。        |
| `channel(index)`                    | `Result<Waveform, AudioError>`      | 提取指定声道。        |
| `resample(sample_rate)`             | `anyhow::Result<Waveform>`          | 重采样。              |
| `slice_ms(start_ms, end_ms)`        | `Waveform`                          | 按毫秒截取。          |
| `split_at_low_energy(max_duration)` | `Result<Vec<Waveform>, AudioError>` | 低能量拆分。          |
| `to_i16_pcm()`                      | `Vec<i16>`                       | 转换为 PCM i16 样本。 |

## Waveform::into_chunks_ms {id="waveform-chunks",tag="method"}

消耗 `Waveform` 并返回惰性 `AudioChunks` 迭代器，适合不再需要完整波形的场景；若仍需
保留原音频，使用借用版本 `chunks_ms()`。

> #### Example
>
> ```rust
> use asr_data::Waveform;
>
> const URL: &str =
>     "https://deepasset.oss-cn-beijing.aliyuncs.com/example.wav";
>
> fn main() -> Result<(), Box<dyn std::error::Error>> {
>     let audio = Waveform::from_url(URL)?;
>     for chunk in audio.into_chunks_ms(100)? {
>         println!("{} ms, final={}", chunk.offset_ms, chunk.is_final);
>     }
>     Ok(())
> }
> ```

| Name            | Description                                             |
| --------------- | ------------------------------------------------------- |
| `self`          | 消耗当前 `Waveform`。                                      |
| `chunk_size_ms` | 每块目标时长，必须大于零。~~u64~~                       |
| **RETURNS**     | 音频块迭代器。~~Result&lt;AudioChunks, AudioError&gt;~~ |

带有原始音频中起始位置的帧对齐音频块。`Waveform::chunks_ms()`、

| `offset_ms`     | chunk 在原始音频中的起始毫秒。~~u64~~           |

| Method                       | Returns                      | Description      |
| ---------------------------- | ---------------------------- | ---------------- |

| `to_i16_pcm()`               | `Vec<i16>`                   | 转换为 PCM i16。 |

| `resample(sample_rate)`      | `anyhow::Result<AudioChunk>` | 重采样。         |

## 音频格式类型 {id="audio-types",tag="types"}

`AudioChannel` 用 `Mono`、`Left`、`Right` 或 `Channel(u16)` 表示文档时间轴的规范
声道。`AudioEncoding` 表示来源编码，支持 `Wav`、`Flac`、`Mp3`、
`Ogg`、`PcmS16Le`、`Other(String)` 和 `Unknown`。

`AudioFormat` 保留解码前的格式信息：

| Field         | Type            | Description  |
| ------------- | --------------- | ------------ |
| `encoding`    | `AudioEncoding` | 音频编码。   |
| `sample_rate` | `u32`           | 原始采样率。 |
| `channels`    | `u16`           | 原始声道数。 |

`AudioFormat::pcm16_mono(sample_rate)` 可便捷创建单声道 PCM S16LE 格式。

## AudioInfo {id="audio-info",tag="struct"}

不含采样数组的音频元信息。

| Field           | Type          | Description               |
| --------------- | ------------- | ------------------------- |
| `sample_rate`   | `u32`         | 原始采样率。              |
| `channels`      | `u16`         | 原始声道数。              |
| `frame_count`   | `u64`         | 每个声道的有效采样帧数。  |
| `source_format` | `AudioFormat` | 原始音频格式。            |

`duration_ms()` 返回浮点毫秒，`timeline_duration_ms()` 返回向上取整后的整数毫秒。

## Audio {id="audio",tag="struct"}

完整音频、时间轴与标注的中心对象，拥有 [`AudioSource`](#audio-source)、
[`AudioInfo`](#audio-info)、各声道 [`Timeline`](#timeline)、业务 metadata 和完整
运行时 [`Waveform`](#waveform)。序列化和数据库持久化会跳过波形。

| Field        | Description                                           |
| ------------ | ----------------------------------------------------- |
| `id`         | 文档 ID。~~String~~                                   |
| `source`     | 原始音频来源。~~AudioSource~~                         |
| `info` | 已探测并持久化的音频信息。~~AudioInfo~~               |
| `metadata`   | 有序 JSON metadata。~~BTreeMap&lt;String, Value&gt;~~ |

timeline 字段通过方法访问，以维持声道、duration 和 audio ID 约束。

| Method | Returns | Description |
| --- | --- | --- |
| `as_waveform()` | `anyhow::Result<Waveform>` | 返回完整波形副本。 |
| `waveform_for_channel(channel)` | `anyhow::Result<Waveform>` | 返回指定声道波形。 |

## Audio::with_id {id="audio-with-id",tag="associated function"}

使用给定 ID 和来源创建记录。构造时探测来源并自动创建各声道 timeline；需要自动 ID
时使用 `Audio::new(source)`。

> #### Example
>
> ```rust
> use asr_data::{Audio, AudioSource};
>
> const URL: &str =
>     "https://deepasset.oss-cn-beijing.aliyuncs.com/example.wav";
>
> fn main() -> anyhow::Result<()> {
>     let doc = Audio::with_id(
>         "example-001",
>         AudioSource::from_url(URL),
>     )?;
>     println!("{}", doc.id);
>     println!("{:.2} ms", doc.info.duration_ms());
>     Ok(())
> }
> ```

| Name        | Description                                                     |
| ----------- | --------------------------------------------------------------- |
| `audio_id`  | 可转换为 `String` 的记录 ID。~~impl Into&lt;String&gt;~~        |
| `source`    | 可转换为 `AudioSource` 的来源。~~impl Into&lt;AudioSource&gt;~~ |
| **RETURNS** | 新音频记录。~~anyhow::Result&lt;Audio&gt;~~                    |

通常使用 `source.load_with_id(...)` 创建完整音频。`Audio::with_id(...)` 是较底层的
探测式构造函数，波形会在第一次调用 `as_waveform()` 时解码。

## AudioStream {id="audio-stream",tag="struct"}

由 [`AudioSource::stream()`](#audio-source-load) 创建的增长中音频文档，与
[`Audio`](#audio) 同级。它拥有相同的 ID、来源、音频信息、metadata 和各声道
timeline，但波形与 timeline 只覆盖已经迭代出的范围。

也可以直接使用 `AudioStream::from_path(...)`、`from_url(...)`、
`from_encoded_bytes(...)`、`from_base64(...)` 或
`from_pcm_s16le(...)` 创建；最后一个参数均为 `chunk_size_ms`。

每次 `next()` 返回 chunk 之前，chunk 样本已经追加到累计波形，各 timeline 也已经
扩展到该 chunk 的结束位置。因此处理当前 chunk 时，可以读取累计波形并修改任意
已接收历史范围的标注。

> #### Example
>
> ```rust
> use asr_data::{AudioChannel, AudioStream};
>
> const URL: &str =
>     "https://deepasset.oss-cn-beijing.aliyuncs.com/example.wav";
>
> fn main() -> anyhow::Result<()> {
>     let mut stream = AudioStream::from_url(URL, 1_000)?;
>
>     while let Some(chunk) = stream.next() {
>         let chunk = chunk?;
>         let timeline = stream.timeline(AudioChannel::Mono)?.unwrap();
>         println!("{}..{} ms / {} ms",
>             chunk.offset_ms,
>             stream.position_ms(),
>             timeline.duration.0,
>         );
>     }
>
>     let audio = stream.into_audio()?;
>     println!("{} frames", audio.info.frame_count);
>     Ok(())
> }
> ```

| Method | Returns | Description |
| --- | --- | --- |
| `timeline(channel)` | `Result<Option<&Timeline>, _>` | 查询增长中的声道时间轴。 |
| `timeline_mut(channel)` | `Result<Option<&mut Timeline>, _>` | 修改已接收范围内的标注。 |
| `as_waveform()` | `Waveform` | 返回累计波形副本。 |
| `position_ms()` | `u64` | 当前已接收结束位置。 |
| `is_complete()` | `bool` | 是否已迭代到来源结尾。 |
| `is_closed()` | `bool` | 是否已提前关闭。 |
| `close()` | `()` | 停止继续迭代。 |
| `into_audio()` | `anyhow::Result<Audio>` | 完整消费后复用全部状态转换为 `Audio`。 |

`into_audio()` 不会重新读取或解码来源，并保留累计波形、timeline、标注和 metadata；
未完整消费或已提前关闭的流不能转换。

## Timeline::annotate_span {id="timeline-annotate-span",tag="method"}

Timeline 唯一的标注写入口。第一个参数 `is_reference` 为 `true` 时写入 reference，
此时 TimeSpan 的 `source` 必须为 `None`；为 `false` 时写入 prediction，此时必须
提供非空 source。内容完全相同会返回已有标注。完整规则见
[标注校验规范](/asr-data/annotations#validation)。

> #### Example
>
> ```rust
> use asr_data::{
>     TimeSpan, Annotation, DurationMs,
>     TimeRange, Timeline, Transcription,
> };
>
> fn main() -> Result<(), asr_data::TimelineSpanError> {
>     let duration = DurationMs(4_205);
>     let mut timeline = Timeline::new("example-001", duration);
>     let transcription = Transcription::new(
>         "甚至出现交易几乎停滞的情况",
>     );
>     let annotation = TimeSpan::new(
>         TimeRange::new(DurationMs(0), duration),
>         Annotation::Transcription(transcription),
>         None,
>     );
>
>     let stored = timeline.annotate_span(true, annotation)?;
>     println!("{}", stored.id);
>
>     let prediction = TimeSpan::new(
>         TimeRange::new(DurationMs(0), duration),
>         Annotation::Transcription(Transcription::new(
>             "甚至出现交易几乎停滞的情况",
>         )),
>         Some("whisper-large".to_owned()),
>     );
>     timeline.annotate_span(false, prediction)?;
>     println!("{:?}", timeline.prediction_sources());
>     Ok(())
> }
> ```

| Name           | Description                                                                            |
| -------------- | -------------------------------------------------------------------------------------- |
| `&mut self`    | 可变借用 timeline。                                                                    |
| `is_reference` | `true` 写 reference，`false` 写 prediction。~~bool~~                                   |
| `annotation`   | 要加入的标注，所有权移入 timeline。~~TimeSpan~~                                        |
| **RETURNS**    | 新标注或已有重复项；source 或重叠无效时报错。~~Result&lt;&TimeSpan, TimelineSpanError&gt;~~ |

## Timeline query methods {id="timeline-query",tag="methods"}

查询、组合转写与管理 prediction source。

> #### Example
>
> ```rust
> use asr_data::{
>     Annotation, DurationMs, TimeRange, TimeSpan, Timeline,
>     Transcription,
> };
>
> fn main() -> Result<(), asr_data::TimelineSpanError> {
>     let duration = DurationMs(4_205);
>     let mut timeline = Timeline::new("example-001", duration);
>     timeline.annotate_span(
>         true,
>         TimeSpan::new(
>             TimeRange::new(DurationMs(0), duration),
>             Annotation::Transcription(Transcription::new(
>                 "甚至出现交易几乎停滞的情况",
>             )),
>             None,
>         ),
>     )?;
>     timeline.annotate_span(
>         false,
>         TimeSpan::new(
>             TimeRange::new(DurationMs(0), duration),
>             Annotation::Transcription(Transcription::new(
>                 "甚至出现交易几乎停滞的情况",
>             )),
>             Some("qwen-asr".to_owned()),
>         ),
>     )?;
>
>     assert_eq!(timeline.span_count(), 2);
>     assert_eq!(
>         timeline.reference_transcript().text,
>         "甚至出现交易几乎停滞的情况"
>     );
>     assert_eq!(
>         timeline.predictions_by_source("qwen-asr").count(),
>         1
>     );
>     timeline.relabel_prediction_source("qwen-asr", "qwen3-asr")?;
>     assert_eq!(timeline.remove_predictions_by_source("qwen3-asr"), 1);
>     Ok(())
> }
> ```

| Method                                 | Returns                                  | Description              |
| -------------------------------------- | ---------------------------------------- | ------------------------ |
| `all_spans()`                          | `impl Iterator<Item = &TimeSpan>`        | 遍历两个集合。           |
| `span_count()`                         | `usize`                                  | 标注总数。               |
| `predictions_by_source(source)`        | `impl Iterator<Item = &TimeSpan>`        | 查询某个预测来源。       |
| `prediction_sources()`                 | `BTreeMap<&str, Vec<&str>>`              | 按类型分组的来源。       |
| `reference_transcript()`               | `Transcript`                             | 组合参考转写。           |
| `prediction_transcript(source)`        | `Transcript`                             | 组合指定来源的预测转写。 |
| `remove_predictions_by_source(source)` | `usize`                                  | 删除来源并返回数量。     |
| `relabel_prediction_source(from, to)`  | `Result<usize, TimelineSpanError>`       | 原子重命名来源。         |
| `validate_spans()`                     | `Result<(), TimelineSpanError>`          | 校验全部标注。           |

`reference_transcript()` / `prediction_transcript()` 会合并
[`Transcription`](#transcription)、[`Sentence`](#sentence)，以及带 transcription 的
[`SpeakerPayload`](#speaker-payload)。
## Timeline::eval {id="evaluation",tag="method"}

按照 reference 评测一个或多个 prediction source。空的 `TimelineEvalConfig::new()` 会
自动发现全部可评测转写与 Activity 来源；显式 builder 只启用请求的任务。

> #### Example
>
> ```rust
> use asr_data::{
>     Timeline, TimelineEvalConfig, TimelineEvalError,
>     TranscriptionNormalization,
> };
>
> fn evaluate(timeline: &Timeline) -> Result<(), TimelineEvalError> {
>     let all = timeline.eval(&TimelineEvalConfig::new())?;
>     println!("{}", all.transcription["qwen-asr"].stats.cer());
>
>     let selected = TimelineEvalConfig::new()
>         .with_transcriptions(["qwen-asr", "whisper"])
>         .with_activity("silero-vad")
>         .with_transcription_normalization(
>             TranscriptionNormalization::None,
>         );
>     let result = timeline.eval(&selected)?;
>     println!("{}", result.activity["silero-vad"].f1());
>     println!(
>         "{}",
>         result.activity["silero-vad"].events["speech"].f1()
>     );
>     Ok(())
> }
> ```

| Config method                              | Description                        |
| ------------------------------------------ | ---------------------------------- |
| `with_transcription(source)`               | 评测一个转写来源。                 |
| `with_transcriptions(sources)`             | 评测多个转写来源。                 |
| `with_all_transcriptions()`                | 自动发现全部转写来源。             |
| `with_activity(source)`                    | 评测一个 Activity 来源。           |
| `with_activity_sources(sources)`           | 评测多个 Activity 来源。           |
| `with_all_activity()`                      | 自动发现全部 Activity 来源。       |
| `with_transcription_normalization(value)`  | 选择中文 TN 或不做标准化。         |

`TimelineEvaluation.transcription` 和 `activity` 都是
`BTreeMap<String, ...Evaluation>`。转写自动发现包含 Transcription、Sentence 及带
transcription 的 Speaker；Activity reference 为空时视为未标注，不参与自动评测。
`ActivityEvaluation.events` 进一步按 event 保存 one-vs-rest 结果；reference 中
event 为 `None` 的范围会参与整体活动评测，并从事件级评测中遮罩。

## 数据集评测 {id="dataset-evaluation",tag="functions"}

`evaluate_dataset()` 聚合任意 Audio 迭代器。需要边读取边处理时可使用
`DatasetEvaluator::new(config)`、`push(&doc)` 和 `finish()`。

> #### Example
>
> ```rust
> use asr_data::{
>     Audio, DatasetEvalError, TimelineEvalConfig,
>     evaluate_dataset,
> };
>
> fn evaluate(docs: &[Audio]) -> Result<(), DatasetEvalError> {
>     let config = TimelineEvalConfig::new();
>     let result = evaluate_dataset(docs.iter(), &config)?;
>     println!("{}", result.transcription["qwen-asr"].cer());
>     println!("{}", result.transcription["qwen-asr"].coverage());
>     Ok(())
> }
> ```

`DatasetEvaluation` 按 source 保存 `DatasetTranscriptionEvaluation` 和
`DatasetActivityEvaluation`。前者累加各 timeline 的 CER 原始计数，后者累加整体
活动与各 event 的 TP、TN、FP、FN 毫秒数；两者都记录参与评测、未标注、缺少预测
的 timeline 数量和 ID，以及 `coverage()`。

## TimeSpan {id="time-span",tag="struct"}

## Annotation {id="annotation",tag="enum"}

`Annotation` 是 `TimeSpan.annotation` 的实际标注内容。相关实现统一位于
`timeline::annotation` 模块，并从 crate root 公开导出。

| Variant                        | Description            |
| ------------------------------ | ---------------------- |
| `Activity(AudioActivity)`      | 声音活动与可选 event。 |
| `Token(Token)`                 | 单个细粒度文本单元。   |
| `Transcription(Transcription)` | 完整转写。             |
| `Sentence(Sentence)`           | 句子级文本片段。       |
| `Speaker(SpeakerPayload)`      | 说话人发话。           |
| `Language(LanguageTag)`        | 语言信息。             |

`confidence()` 从 Activity、Token、Transcription、Speaker 读取可选置信度；
[`Sentence`](#sentence) 与 [`Language`](#language-tag) 始终返回 `None`。

## AudioActivity {id="audio-activity",tag="struct"}

声音活动 payload。`event: Option<String>` 用于保存可选的事件类别；`None` 表示只确
认区间内存在活动，但类别未知。

> #### Example
>
> ```rust
> use asr_data::AudioActivity;
>
> fn main() {
>     let speech = AudioActivity::new()
>         .with_event("speech")
>         .with_confidence(0.98);
>     let unknown = AudioActivity::new();
>     assert_eq!(speech.event.as_deref(), Some("speech"));
>     assert_eq!(unknown.event, None);
> }
> ```

添加到 Timeline 时，空白 event 会被拒绝。相同 event 的 Activity 不能在同一
reference 集合或同一 prediction source 中重叠；不同 event 可以重叠。

| Field        | Description                               |
| ------------ | ----------------------------------------- |
| `event`      | 可选活动类别。~~Option&lt;String&gt;~~    |
| `confidence` | 可选活动检测置信度。~~Option&lt;f32&gt;~~ |

## Transcription {id="transcription",tag="struct"}

完整转写 payload，可以包含 [`Token`](#token) 列表。`Transcription::new(text)` 提
供合理默认值；再按需修改公开字段是 Rust 中最直接的构造方式。

> #### Example
>
> ```rust
> use asr_data::{DurationMs, TimeRange, Token, Transcription};
>
> fn main() {
>     let mut transcription = Transcription::new(
>         "甚至出现交易几乎停滞的情况",
>     );
>     transcription.language = Some("zh".to_owned());
>     transcription.confidence = Some(0.93);
>     transcription.tokens = vec![
>         Token::new("甚至出现").with_range(TimeRange::new(
>             DurationMs(0),
>             DurationMs(1_800),
>         )),
>         Token::new("交易几乎停滞的情况").with_range(TimeRange::new(
>             DurationMs(1_800),
>             DurationMs(4_205),
>         )),
>     ];
>
>     println!("{}", transcription.text);
> }
> ```

| Field        | Description                             |
| ------------ | --------------------------------------- |
| `text`       | 完整转写文本。~~String~~                |
| `tokens`     | 细粒度 token。~~Vec&lt;Token&gt;~~      |
| `language`   | 可选语言标签。~~Option&lt;String&gt;~~  |
| `confidence` | 可选转写级置信度。~~Option&lt;f32&gt;~~ |

## Sentence {id="sentence",tag="struct"}

句子级文本片段。与 [`Transcription`](#transcription) 类似，但不携带
`confidence`；组合 transcript 时会与 Transcription、Speaker 内转写一起按开始时间
排序合并。

> #### Example
>
> ```rust
> use asr_data::{
>     Annotation, DurationMs, Sentence, TimeRange, TimeSpan,
>     Timeline, Token,
> };
>
> fn main() -> Result<(), asr_data::TimelineSpanError> {
>     let mut sentence = Sentence::new("甚至出现交易几乎停滞的情况");
>     sentence.language = Some("zh".to_owned());
>     sentence.tokens = vec![Token::new("甚至出现").with_range(
>         TimeRange::new(DurationMs(0), DurationMs(1_800)),
>     )];
>
>     let mut timeline = Timeline::new("example-001", DurationMs(4_205));
>     timeline.annotate_span(
>         true,
>         TimeSpan::new(
>             TimeRange::new(DurationMs(0), DurationMs(4_205)),
>             Annotation::Sentence(sentence),
>             None,
>         ),
>     )?;
>     assert_eq!(
>         timeline.reference_transcript().text,
>         "甚至出现交易几乎停滞的情况"
>     );
>     Ok(())
> }
> ```

| Field      | Description                            |
| ---------- | -------------------------------------- |
| `text`     | 句子文本。~~String~~                   |
| `tokens`   | 细粒度 token。~~Vec&lt;Token&gt;~~     |
| `language` | 可选语言标签。~~Option&lt;String&gt;~~ |

当前重叠校验不约束 `Sentence`；它仍会进入 transcript 组合与转写评测来源发现。
Python 目前不能新建 `Sentence`，但可以从已有数据读出。
## SpeakerPayload {id="speaker-payload",tag="struct"}

一次说话人发话，由名称和可选 [`Transcription`](#transcription) 组成。

> #### Example
>
> ```rust
> use asr_data::{SpeakerPayload, Transcription};
>
> fn main() {
>     let mut speaker = SpeakerPayload::new("agent").with_confidence(0.96);
>     speaker.transcription = Some(Transcription::new(
>         "甚至出现交易几乎停滞的情况",
>     ));
>
>     let text = speaker
>         .transcription
>         .as_ref()
>         .map(|value| value.text.as_str())
>         .unwrap_or_default();

> }
> ```

| Field           | Description                                         |
| --------------- | --------------------------------------------------- |
| `name`          | 说话人名称或业务 ID。~~SpeakerId~~                  |
| `transcription` | 该次发话的完整转写。~~Option&lt;Transcription&gt;~~ |
| `confidence`    | 可选说话人识别置信度。~~Option&lt;f32&gt;~~         |

## LanguageTag {id="language-tag",tag="type"}

语言标签类型别名，等价于 `String`。通过 `Annotation::Language` 挂到
[`TimeSpan`](#time-span) 上，用于保存区间级语言信息；不参与 transcript 组合，也
没有 payload 级置信度。

> #### Example
>
> ```rust
> use asr_data::{
>     Annotation, DurationMs, TimeRange, TimeSpan, Timeline,
> };
>
> fn main() -> Result<(), asr_data::TimelineSpanError> {
>     let mut timeline = Timeline::new("example-001", DurationMs(4_205));
>     timeline.annotate_span(
>         true,
>         TimeSpan::new(
>             TimeRange::new(DurationMs(0), DurationMs(4_205)),
>             Annotation::Language("zh".to_owned()),
>             None,
>         ),
>     )?;
>
>     let language = timeline.reference.iter().find_map(|span| {
>         match &span.annotation {
>             Annotation::Language(tag) => Some(tag.as_str()),
>             _ => None,
>         }
>     });
>     assert_eq!(language, Some("zh"));
>     Ok(())
> }
> ```

| Name        | Description                                      |
| ----------- | ------------------------------------------------ |
| **ALIAS**   | `type LanguageTag = String`                      |
| **USAGE**   | `Annotation::Language(LanguageTag)`              |

Python 目前不能新建 `Language` annotation，但可以从已有数据读出。

## AudioDb {id="audio-db",tag="struct"}

| Method             | Description                                         |
| ------------------ | --------------------------------------------------- |
| `create(path)`     | 创建新数据库。~~Result&lt;AudioDb, AudioDbError&gt;~~ |

数据库查询参数。通常从 `AudioQuery::default()` 开始，只覆盖所需字段。

>
>     let mut doc = Audio::with_id("example-001", AudioSource::from_url(URL))?;
>     doc.metadata.insert("split".into(), serde_json::json!("train"));
>     db.insert(&doc)?;
>
>     let mut query = AudioQuery::default();
>     query.limit = 100;
>     query.created_from = Some(SystemTime::now() - Duration::from_secs(7 * 24 * 60 * 60));
>     query.updated_from = Some(SystemTime::now() - Duration::from_secs(24 * 60 * 60));
>     query.metadata.insert("split".into(), serde_json::json!("train"));
>     let page = db.query(&query)?;
>     println!("{} result", page.len());
>
>     drop(db);
>     fs::remove_file(path)?;
>     Ok(())
> }
> ```

此示例需要在 `Cargo.toml` 中同时依赖 `serde_json`。

| Field           | Description                                            |
| --------------- | ------------------------------------------------------ |
| `limit`         | 最大返回数，默认 `DEFAULT_QUERY_LIMIT`。~~usize~~      |
| `after`         | 上一页最后一个 audio ID。~~Option&lt;String&gt;~~      |
| `min_duration`  | 可选最短时长。~~Option&lt;DurationMs&gt;~~             |
| `max_duration`  | 可选最长时长。~~Option&lt;DurationMs&gt;~~             |
| `created_from`  | 创建时间下界，包含。~~Option&lt;SystemTime&gt;~~       |
| `created_until` | 创建时间上界，不包含。~~Option&lt;SystemTime&gt;~~     |
| `updated_from`  | 修改时间下界，包含。~~Option&lt;SystemTime&gt;~~       |
| `updated_until` | 修改时间上界，不包含。~~Option&lt;SystemTime&gt;~~     |
| `metadata`      | 精确匹配的 metadata。~~BTreeMap&lt;String, Value&gt;~~ |

`AudioDb::eval(&query, &config)` 使用同一组筛选条件，并自动用 `query.limit` 作为每
批读取数量遍历全部匹配文档：

> #### Example
>
> ```rust
> use asr_data::{
>     AudioDb, AudioQuery, DatasetEvalError, TimelineEvalConfig,
> };
>
> fn evaluate(
>     db: &AudioDb,
>     query: &AudioQuery,
> ) -> Result<(), DatasetEvalError> {
>     let result = db.eval(
>         query,
>         &TimelineEvalConfig::new().with_transcription("qwen-asr"),
>     )?;
>     println!("{}", result.transcription["qwen-asr"].cer());
>     Ok(())
> }
> ```

`limit` 不得超过 `MAX_QUERY_LIMIT`（10,000），且时长或时间范围的下界不得大于上界
。创建和修改时间由数据库自动维护，不会写入 `Audio`。

## 时间类型 {id="time-types",tag="structs"}

Rust API 用 newtype 表示时间和采样点，避免把不同单位的整数混用。

| Type                       | Description                             |
| -------------------------- | --------------------------------------- |
| `DurationMs(pub u64)`      | 毫秒时长；`seconds()` 转换为 `f64` 秒。 |
| `SampleIndex(pub u64)`     | 采样点索引。                            |
| `TimeRange { start, end }` | 半开时间范围。                          |

`TimeRange::duration()` 返回饱和相减后的时长；`overlaps(&other)` 判断范围重叠；
`contains(point)` 使用 `start <= point < end`。

## 中文文本归一化 {id="normalize-zh",tag="function"}

`normalize_zh()` 使用内嵌的中文 TN FST，把书写形式转换为口语形式。资源随 crate
一起编译，不需要安装或调用外部服务。

> #### Example
>
> ```rust
> use asr_data::normalize_zh;
>
> fn main() -> Result<(), Box<dyn std::error::Error>> {
>     let text = normalize_zh("2024年")?;
>     assert_eq!(text, "二零二四年");
>     Ok(())
> }
> ```

| Name        | Description                                                  |
| ----------- | ------------------------------------------------------------ |
| `text`      | 需要归一化的文本。~~&str~~                                   |
| **RETURNS** | 归一化结果。~~Result&lt;String, TextNormalizationError&gt;~~ |

## CER {id="cer",tag="functions"}

计算字符错误率。

> #### Example
>
> ```rust
> use asr_data::{compute_cer, normalize_for_cer};
>
> fn main() {
>     let reference = normalize_for_cer(
>         "甚至出现交易几乎停滞的情况",
>         true,
>     );
>     let hypothesis = normalize_for_cer(
>         "甚至出现交易几乎停止的情况",
>         true,
>     );
>     let stats = compute_cer(&reference, &hypothesis);
>     println!("CER: {:.2}%", stats.cer() * 100.0);
> }
> ```

| Function                                 | Returns    | Description                          |
| ---------------------------------------- | ---------- | ------------------------------------ |
| `normalize_for_cer(text, remove_spaces)` | `String`   | 移除标点，并按参数决定是否移除空白。 |
| `compute_cer(reference, hypothesis)`     | `CerStats` | 计算编辑距离统计。                   |

## 错误类型 {id="errors",tag="enums"}

公共错误均实现 `std::error::Error`，应用层可以使用 `?` 传播或按 enum variant 精
确匹配。

| Type                      | Scope                                      |
| ------------------------- | ------------------------------------------ |
| `AudioError`              | 样本、声道、PCM 和 chunk 校验。            |
| `AudioDbError`            | SQLite、序列化、查询和迁移。               |
| `AudioChannelError`       | 非规范声道。                               |
| `AudioTimelineError`      | timeline duration 创建约束。               |
| `AudioValidationError`    | Audio 完整性校验。                         |
| `TimelineSpanError`       | 标注 source、activity event 与重叠冲突。   |
| `TimelineEvalError`       | 单条 timeline 评测配置或执行错误。         |
| `DatasetEvalError`        | 数据集评测聚合错误。                       |
| `TextNormalizationError`  | 中文 TN FST 归一化失败。                   |

`TimelineSpanError` 常见 variant 包括 `ReferenceHasSource`、
`PredictionMissingSource`、`InvalidPredictionSource`、`InvalidActivityEvent` 和
`Overlap`。完整定义以本地 crate 源码与生成的 rustdoc 为准。
