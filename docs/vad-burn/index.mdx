`vad-burn` 是一个 Rust 和 Python 可用的 VAD 推理库，面向 16 kHz 单声道
归一化浮点 PCM。它使用 Burn Flex CPU 后端，包含 FSMN VAD 和 FireRedVAD
两条模型路径，都支持离线整段检测和流式检测。

源码：[di-osc/vad-burn](https://github.com/di-osc/vad-burn)

## 安装 {id="installation"}

Python 用户直接安装 wheel：

```bash
pip install vad-burn
```

Rust 项目添加 crate：

```bash
cargo add vad-burn
```

Apple Silicon 上可以选择启用 `apple-amx` feature：

```toml
[dependencies]
vad-burn = { version = "0.1.3", features = ["apple-amx"] }
```

## 核心概念 {id="concepts"}

- **Waveform**：Rust API 使用 [`Waveform`](/vad-burn/api#waveform) 表示输入音频。
  Python API 直接接收 `Sequence[float]` 和 `sample_rate`。样本约定均为
  `[-1.0, 1.0]` 范围内的单声道浮点 PCM。
- **VadOptions**：Rust [`VadOptions`](/vad-burn/api#vad-options) 与 Python
  [`VadOptions`](/vad-burn/api-python#vad-options) 控制阈值、最短语音、静音闭合
  时长和最长分段；`pad_ms` 当前仅为兼容而保留。FSMN 和 FireRed 共用同一组选项，
  但 `max_segment_ms=0` 的回退行为不同，详见对应 API 参考。
- **离线检测**：`detect()` 一次处理完整音频并返回
  `VadSegment` 列表（[Rust](/vad-burn/api#vad-segment) /
  [Python](/vad-burn/api-python#vad-segment)）。
- **带计时检测**：`detect_with_timing()` 返回 segments、帧级分数和前端/前向/
  后处理计时，适合验证性能。
- **流式检测**：`new_stream()` 创建有状态 stream；反复 `push()` 输入 chunk，
  最后调用 `finish()` 刷出剩余片段。
- **模型来源**：`from_modelscope()` 下载并复用 ModelScope 缓存；
  `from_pretrained()` 加载本地模型目录。

## 快速开始 {id="quickstart"}

Rust：

```rust
use vad_burn::{FsmnVadModel, VadOptions, Waveform};

fn main() -> anyhow::Result<()> {
    let model = FsmnVadModel::from_modelscope()?;
    let waveform = Waveform::new(samples, 16_000);
    let segments = model.detect(&waveform, &VadOptions::default())?;

    for segment in segments {
        println!(
            "{}..{} ms {:.3}",
            segment.range.start.0,
            segment.range.end.0,
            segment.probability
        );
    }
    Ok(())
}
```

Python：

```python
from vad_burn import FsmnVadModel, VadOptions

vad = FsmnVadModel.from_modelscope()
segments = vad.detect(samples, 16000, VadOptions())

for segment in segments:
    print(segment.start_ms, segment.end_ms, segment.probability)
```

## CPU 加速 {id="acceleration"}

`FsmnVadModel` 和 `FireRedVadModel` 都使用 Burn Flex CPU 后端。默认构建启用 SIMD
和 Rayon；Apple Silicon 上还可以启用 `apple-amx` feature，让 Burn Flex 使用其
Apple AMX 支持。当前 crate 没有 Metal/GPU feature，Python wheel 也使用 CPU
后端。
