# Libraries

`di-osc` 开源库文档站。基于 spaCy 站点架构的 Next.js 静态站；文档内容在 `docs/`，导航在 `meta/`。

## 文档规范

- 项目的文档都应该遵循循序渐进的原则，一般现有一个快速开始，介绍项目的基本概念和使用方法。然后在逐步介绍项目的核心概念，详细功能等，必要的时候也要给出图片作为辅助说明。
- 保持主页面文档，左侧边栏导航，右侧边栏示例的布局。
- 示例要尽量完整，可以是代码也可以是CLI命令，配置参数等示例。示例应该在小节的描述之后，且不应太长导致跟后面的示例在视觉上连在一起。
- 每个项目的文档最后都要给出完整的API，如果是多语言的，要给出所有语言的API，且每个API尽量给出示例，如果是属性等简单内容，可以不给出示例。
